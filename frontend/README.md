# Karyolab v2 — Журнал (фронтенд-прототип)

Кликабельный фронтенд-прототип раздела **Журнал** Karyolab v2. Без бэкенда, на
мок-данных. Назначение — согласовать с учёными расположение всех экранов, форм и
карточек.

## Стек

- React 18 + Vite + TypeScript
- Tailwind CSS (палитра из `docs/журнал/12_дизайн_журнала.md`, шрифт Inter)
- React Router v6
- Zustand (мок-стор в памяти)
- date-fns, lucide-react, react-hot-toast, clsx

## Локальный запуск

```bash
cd karyolab_v2/frontend
npm ci          # или npm install
npm run dev
```

После запуска: <http://localhost:5173> — автоматически редиректит на `/журнал`.

Дополнительные команды:

```bash
npm run build   # production-сборка в dist/
npm run preview # локально посмотреть собранный dist/
```

## Запуск через Docker

В каталоге `karyolab_v2/frontend` уже лежит `Dockerfile` и `nginx.conf.template`:
multi-stage сборка на Node 20, готовый `dist/` отдаёт nginx, порт берётся из
переменной окружения `PORT`.

```bash
cd karyolab_v2/frontend
docker build -t karyolab-journal .
docker run --rm -p 8080:8080 -e PORT=8080 karyolab-journal
```

Открыть <http://localhost:8080> → `/журнал`.

Healthcheck: `GET /healthz` → `200 ok`.

## Деплой на Railway

Railway сам обнаружит `Dockerfile` в `karyolab_v2/frontend`. Достаточно:

1. Создать новый сервис **Empty Service → Deploy from a GitHub repo** (или
   `railway up` из CLI), указав корнем сервиса каталог `karyolab_v2/frontend`.
2. Railway автоматически прокидывает свою переменную `PORT` — Dockerfile уже её
   уважает (envsubst в `nginx.conf.template`).
3. Healthcheck path в настройках сервиса можно поставить `/healthz`.
4. После деплоя зайти на присвоенный домен — фронт сразу откроется на
   `/журнал`.

> Если деплоим не из CLI/GitHub, а вручную — `docker build` + `docker push` в
> любой реестр и далее «Deploy from image» в Railway. Переменная `PORT`
> подставится автоматически.

## Демо-сценарий для учёных

1. **Главная** (`/журнал`)
   - календарь текущего месяца; выбранный день подсвечен мятным; кнопка
     «Сегодня» возвращает к сегодняшней дате;
   - длинные плашки — проращивание/гибридизация (многодневные), короткие —
     отмывка / фотографирование / свободный ивент;
   - правая колонка: «События дня» (вместе с заметками, оставленными в этот
     день), «Заметки» (большая активная + 2 свёрнутые) и тёмно-зелёный
     «Счётчик тильта»;
   - блок **Прогресс / поиск висяков** — колонки по шагам жизненного цикла
     (`Созрели · нет препарата` → `Препарат создан` → `Отмыт` → `Гибридизован`
     → `Сфотографирован` → `Результат`).
2. **Клик на день** в календаре только переключает «События дня». Создание
   ивента — по `+` в углу ячейки или через большую кнопку «+ Добавить ивент».
3. **`+ Добавить ивент`** → переключатель типа ивента, далее одна из 6 форм:
   проращивание, создание препарата, предгибридизационная отмывка,
   гибридизация, фотографирование, свободный ивент.
4. **`+ Добавить образец`** в боковой панели → форма регистрации образца с
   проверкой конфликта по ID.
5. **Карточка ивента** показывает «Что изменилось» — счётчики, ссылки на
   образцы и кнопки следующего логичного шага.
6. **Карточка образца** — `S-1818.25` или любой ID из мок-данных:
   ```
   /журнал/образец/1818.25
   ```
   Внутри — дерево «Растения → Препараты → Окраски» с разворачиваемой веткой
   **Смесь растений** и историей циклов окраски.
7. **Архив заметок** — кнопка «Архив» в виджете заметок или
   `/журнал/архив`. Слева — заметки по месяцам с фильтрами по тегам и поиском,
   справа — `История тильта`.
8. **Тильт `+`** — увеличивает счётчик, показывает тост «Тильт +1».

## Маршруты

- `/` → редирект на `/журнал`
- `/журнал` — главная
- `/журнал/новый-ивент?type=…&date=…&prep=…` — создание ивента (с
  предзаполнением даты и/или выбранных препаратов)
- `/журнал/новый-образец` — регистрация образца
- `/журнал/ивент/:id` — карточка ивента (тип определяет рендер)
- `/журнал/образец/:id` — карточка образца
- `/журнал/архив` — архив заметок и история тильта
- `/кариотип`, `/кариотип/:sub`, `/атлас` — заглушки «Раздел в разработке»

обработка изображений, бизнес-логика статусов и сами разделы Кариотип/Атлас.
