# Ивенты

Ивент в журнале - это пользовательское лабораторное действие, которое видно в календаре, ленте событий и карточках связанных объектов. Важное правило: после ручного создания образца новые лабораторные объекты и изменения статусов должны происходить через добавление ивентов.

Сам `образец` создается не ивентом, а отдельной формой `Добавить образец`. Ивенты уже работают с существующими образцами: добавляют их в проращивание, создают препараты, отмывают, гибридизуют и закрывают циклы фотографированием.

Ивент не должен быть голой заметкой в базе. Он должен отвечать на вопросы:

- что произошло;
- когда произошло;
- какие образцы, растения, препараты или окраски участвовали;
- какие объекты созданы;
- какие статусы изменены;
- где теперь лежит материал;
- какой следующий шаг стал доступен.

## Общие Поля Ивента

Каждый ивент имеет:

- тип;
- название;
- дату и, если нужно, время начала;
- дату и время окончания для длительных событий;
- список связанных образцов;
- список связанных препаратов или окрашенных препаратов, если применимо;
- комментарий;
- отображение в календаре;
- отображение в ленте выбранного дня;
- ссылку на собственную карточку.

Ивент можно создать из кнопки `Add Event`, из пустой даты календаря или из быстрого действия в блоке прогресса. Если пользователь пришел из прогресса или карточки образца, форма должна открываться уже с выбранным объектом: образцом, препаратом или окраской.

После сохранения ивента интерфейс должен показать, что именно изменилось: сколько препаратов создано, сколько отмыто, какие окраски появились, сколько препаратов помечены как выброшенные или снова доступные для работы.

## Типы Ивентов

Основные типы:

- `проращивание`;
- `создание препарата`;
- `отмывка препаратов`;
- `гибридизация`;
- `фотографирование`;
- `свободный ивент`.

## Проращивание

Проращивание - самый длинный ивент. Он начинается с закладки семян и продолжается через весь протокол до созревания и хранения.

Особенности:

- может включать несколько образцов сразу;
- отображается в календаре длинной плашкой от начала до конца;
- внутри содержит под-ивенты;
- первые динамические этапы редактируемы;
- от даты переноса в термостат дальше идет жесткий временной каркас.

Название по умолчанию: `проращивание партия дата начала`.

Проращивание подробно описано в [05_проращивание_и_протоколы.md](05_проращивание_и_протоколы.md).

## Создание Препарата

Создание препарата - индивидуальный ивент. Он создает объект `препарат` от конкретного растения конкретного образца.

Поля формы:

- образец;
- растение;
- качество препарата;
- место хранения;
- комментарий.

Если растения еще нет, пользователь должен иметь возможность создать его в этом сценарии или выбрать режим, при котором растение создается автоматически.

Практически форма должна позволять быстро создать несколько препаратов подряд от одного образца или растения. При этом каждый препарат все равно должен получить собственный ID, качество и место хранения, чтобы потом его можно было выбрать в отмывке.

Отображение:

- в календаре это плашка `создание препарата`;
- если за день создано несколько препаратов, календарь может показывать одну агрегированную плашку с количеством;
- карточка ивента показывает список препаратов, качество и места хранения.

Результат:

- создан новый препарат;
- препарат получает статус `создан`;
- образец переходит в `в работе`, если еще не был там.

## Отмывка Препаратов

Отмывка - групповой ивент. Он выбирает несколько существующих препаратов и меняет их статус и место хранения.

Поля формы:

- список препаратов;
- дата проведения;
- новое место хранения;
- возможность применить место хранения ко всем выбранным;
- индивидуальные места хранения при необходимости;
- протокольный комментарий.

В списке выбора должны показываться только препараты, которые можно отмывать: созданные препараты без отмывки или препараты после фотографирования, выбранные для повторной работы. Выброшенные препараты и активные окрашенные препараты не должны попадать в список как доступные.

Название по умолчанию: `отмывка партия дата проведения`.

Результат:

- выбранные препараты получают статус `отмыт`;
- обновляется место хранения;
- событие видно в календаре, ленте дня и карточках образцов;
- карточка ивента показывает таблицу отмытых препаратов.

## Гибридизация

Гибридизация - групповой ивент для отмытых препаратов. Она создает для каждого выбранного препарата объект `окрашенный препарат`.

Поля формы:

- список отмытых препаратов;
- дата операции;
- название партии;
- зонды по каналам;
- флюорохромы, если они известны;
- комментарий.

Зонды должны выбираться по одному, а не только готовой строкой панели. Например:

- красный канал: `pAs1 + pAs119`;
- зеленый канал: `GAA`;
- синий канал: `DAPI`.

Форма должна показывать счетчик цикла окраски по каждому выбранному препарату: `окраска 1`, `окраска 2` или `окраска 3`. Если препарат уже прошел три окраски, он не должен быть доступен для новой гибридизации без отдельного ручного решения.

Название по умолчанию: `гибридизация зонды партия дата проведения`.

Длительность: 2 дня.

Результат:

- у каждого выбранного препарата создается новый цикл окраски;
- физический препарат получает статус `гибридизован`;
- окрашенный препарат получает статус `создан`;
- гибридизация отображается в календаре как длительный ивент.

## Фотографирование

Фотографирование - ивент по окрашенным препаратам. Практически оно часто выполняется индивидуально, но в интерфейсе может агрегироваться за день, чтобы не создавать четыре отдельные плашки.

Поля формы:

- список доступных окрашенных препаратов;
- дата фотографирования;
- для каждого выбранного препарата дальнейшая судьба:
  - `отмыт` - стекло можно использовать для следующей гибридизации;
  - `выброшен` - стекло закрыто;
- новое место хранения для отмытых препаратов;
- комментарий.

Результат:

- окрашенный препарат получает статус `сфотографирован`;
- физический препарат получает дальнейший статус `отмыт после фото` или `выброшен`;
- событие появляется в календаре и ленте;
- пользователь может импортировать или связать фото в разделе кариотипа.

Важно: фотографирование не должно быть отдельной `партией ради партии`. Смысл этого экрана - закрыть текущую окраску и решить судьбу физического препарата. Фото и PSD дальше прикрепляются к конкретной гибридизации в разделе кариотипа.

## Свободный Ивент

Свободный ивент - ручная заметка календаря. Он нужен для событий, которые не меняют статус лабораторных объектов, но важны для истории.

Поля:

- название;
- дата;
- заметка;
- необязательное вложение.

Свободный ивент отображается в календаре и ленте событий. Он отличается от заметок главной страницы: заметки - это отдельный блок без обязательного отображения в календаре.

Свободный ивент полезен для наблюдений, инцидентов, внешних действий и напоминаний, которые важны в хронологии, но не должны менять статус образцов или препаратов.

## Доступность Типов Ивентов

Форма создания ивента должна помогать пользователю не ошибиться:

- `создание препарата` доступно, когда выбран образец и растение или можно создать растение на месте;
- `отмывка` предлагает только препараты, которые физически можно отмыть;
- `гибридизация` предлагает только отмытые препараты с доступным циклом окраски;
- `фотографирование` предлагает только активные окрашенные препараты;
- `свободный ивент` доступен всегда.

Если подходящих объектов нет, форма не должна показывать пустую таблицу без объяснения. Лучше показать короткую подсказку: например, `Нет отмытых препаратов. Сначала создайте ивент отмывки`.

## Карточка Ивента

У каждого ивента есть карточка:

- название и дата;
- тип;
- статус выполнения;
- список образцов;
- список препаратов, если есть;
- список зондов, если есть;
- места хранения до и после, если есть;
- комментарий;
- ссылки на карточки связанных объектов.
