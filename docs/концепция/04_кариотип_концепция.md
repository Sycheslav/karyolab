# Кариотип — Концепция

Раздел `Кариотип` — это рабочая мастерская: импорт PSD, разметка хромосом, создание идеограмм, сборка кариотипа, экспорт.

## Структура Раздела

В боковом меню `Кариотип` — **один пункт**. Внутри он раскрывается на **горизонтальные вкладки** (не подпункты в меню):

- **Импорт** — загрузка PSD, привязка к окрашенному препарату, извлечение хромосом;
- **Кариотип** — разметка хромосомы и разметка генома;
- **Экспорт** — сборка обзорных изображений и таблиц данных.

Подразделы вынесены в верхние вкладки намеренно — левое меню должно быть минимальным, чтобы холст разметки занимал максимум пространства.

## Импорт PSD

Оператор:

1. Перетаскивает PSD-файл, который пришёл от микроскопа.
2. Видит карточку активного выбора (образец / окрашенный препарат / зонды / даты), чтобы убедиться, что импорт идёт в правильное место.
3. Видит список найденных слоёв с миниатюрами и временными именами `untitled-1`, `untitled-2`, ... — это имя файла до этапа разметки.
4. Может исключить служебные слои (фон, мусор) до commit'а.
5. Нажимает `Сохранить N хромосом`.

После импорта **каждой хромосоме сразу присваивается канонический ID** `<метафаза>.cNN`. Этот ID **не меняется**, даже если оператор позже переименует файл хромосомы при разметке.

## Разметка Хромосомы

Оператор размечает каждую хромосому отдельно. Шкалы располагаются рядом с изображением хромосомы в одинаковом масштабе по высоте:

- **размер и центромера** — одна шкала с **тремя ползунками** (верхняя точка размера, нижняя точка размера, центромера);
- **красный канал** — все красные сигналы;
- **зелёный канал** — все зелёные сигналы;
- **аномалии** — отдельная линия рядом со шкалами сигналов.

### Типы Сигналов

Ровно **четыре**:

- маленькая точка;
- точка;
- большая точка;
- отрезок.

На итоговой идеограмме сигнал визуально выглядит как **две точки рядом на одном уровне** (или два отрезка для типа `отрезок`) — так же, как сигнал виден на фотографии.

### Имя Зонда (Опционально)

При добавлении сигнала можно указать имя зонда из справочника, но это **не обязательно**. Оператор не должен учить весь справочник, чтобы что-то сохранить.

### Хромосомные Аномалии

При добавлении аномалии открывается поиск по **справочнику хромосомных мутаций** в атласе. Если нужного типа нет — оператор добавляет его сам. Программа не предлагает свой набор по умолчанию.

### Класс И Субгеном

Класс и субгеном — это **одно понятие**. Запись `5D` обозначает одновременно класс `5` и субгеном `D`. На этапе разметки идеограммы оператор может задать класс/субгеном или оставить пустым (имя файла останется `untitled-N` до этапа разметки генома).

При задании класса имя файла превращается в осмысленное (например, `1730.25.1.1.1.m1.c01_5D.png`). Канонический ID не меняется.

## Разметка Генома

Оператор раскладывает хромосомы по матрице **класс × субгеном**.

### Гибкие Шаблоны Субгеномов

В атласе есть готовые шаблоны:

- пшеница (A, B, D);
- тритикале (A, B, D, R);
- рожь (R);
- эгилопс (U, M, ...);
- пользовательские — оператор может создать, отредактировать, удалить любой шаблон.

При замещениях и чужеродном материале добавляется новый субгеном на лету.

### Кнопка "Разложить По Классам"

Кнопка в UI называется именно так (не "auto-sort"). Программа **только предлагает** расстановку:

- для нормальных хромосом — по сигналам и параметрам идеограммы;
- для аномальных хромосом (трисомии, моносомии, нуллисомии, замещения) — **программа сама ничего не решает**. Все такие записи проставляет оператор вручную.

### Готовность Кариотипа

Готовность определяется по плоидности из настроек вида в атласе. Например, для **гексаплоидной пшеницы — 42 хромосомы**, не 21 (пара генов даёт две хромосомы).

### Два Уровня

Разметка генома работает на двух уровнях:

- **уровень метафазы** — формирует **кариотип метафазы** для одной конкретной метафазы;
- **уровень гибридизации** — формирует **кариотип образца** из лучших хромосом всех метафаз гибридизации.

## Кариотип Метафазы И Кариотип Образца

Это **разные сущности**.

| Признак | Кариотип метафазы | Кариотип образца |
|---|---|---|
| Источник | одна метафаза | лучшие хромосомы всех метафаз гибридизации |
| ID | `<метафаза>.kar.<n>` | `<образец>.kar.<n>` |
| Сколько у образца | по одному на каждую размеченную метафазу | один или несколько (разные растения, гибридизации) |
| Где видно | карточка образца, отдельный список | карточка образца, основные плитки кариотипов |
| Когда экспортируется | для технических обзоров | для публикаций и отчётов |

При сохранении **кариотипа образца** программа **копирует файлы выбранных хромосом** в его собственную папку. Это нужно, чтобы:

- кариотип не сломался, если в исходной метафазе хромосому удалили;
- можно было заменить хромосому в утверждённом кариотипе без отзыва утверждения.

## Статусы Кариотипа

Кариотип образца имеет четыре статуса:

- `черновик` — оператор работает;
- `утверждён` — оператор подтвердил, можно экспортировать;
- `экспортирован` — есть хотя бы один экспорт;
- `архивный` — устаревший, скрыт из активной работы.

Нет статусов "неполный" или "готов к проверке" — оператор сам решает, когда кариотип готов.

## Экспорт

Оператор выбирает источник:

- кариотип образца;
- кариотип метафазы;
- эталонный (теоретический) кариотип из атласа;
- свободная таблица из произвольного набора образцов.

Шаблоны:

- стандартный обзор (один кариотип);
- сравнение нескольких;
- свободная таблица — оператор сам настраивает строки и столбцы.

### Общее Правило

В любом шаблоне:

- хромосомы одного класса всегда рядом;
- хромосомы выравниваются по центромере (кроме хромосом без размеченной центромеры — они показываются с предупреждением).

### Режимы Отображения

- только хромосома (фото без фона метафазы);
- только идеограмма;
- хромосома + идеограмма рядом;
- хромосома сверху, идеограмма снизу.

### Подписи

- название образца;
- путь к метафазе (если включено);
- цвета каналов: текст красным/зелёным/синим;
- заголовок субгенома и номер класса;
- подписи аномалий со стрелками к участку — **не перекрывают** хромосому или идеограмму.

### Форматы

- **TIFF** для изображений (максимальное качество, без ползунка);
- **Excel/текст** для таблиц данных (не картинка таблицы).
