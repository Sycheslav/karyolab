# Рабочие Сценарии

Типовые маршруты пользователя в Karyolab v2. Каждый сценарий описан простыми шагами без технических деталей.

## Сценарий 1. Полный Цикл: От Засева До Экспорта

1. **Журнал → Создать образец**. Оператор заводит образец `1730.25` (имя, вид, родители, год).
2. **Журнал → Засев**. Оператор отмечает дату засева. Программа показывает на календаре полупрозрачный блок "перенос в холодильник → 2 недели до созревания".
3. **Журнал → Создание препарата**. Когда растения созрели, оператор делает препарат от растения или смеси растений. Препарат `1730.25.1.1` появляется в журнале со статусом `создан`.
4. **Журнал → Отмывка**. Оператор отмывает партию препаратов. Препарат переходит в `отмыт`. Отмывка имеет имя `WP-{YYYY-MM-DD}` по умолчанию.
5. **Журнал → Гибридизация**. Оператор выбирает препараты и зонды из справочника. Каналы подтягиваются из флюорохромов зондов, DAPI автоматически синий. Создаются окрашенные препараты `1730.25.1.1.1`. Препарат становится `гибридизован`.
6. **Журнал → Фотографирование**. Под микроскопом получены фото. Оператор для каждого препарата выбирает судьбу: `сфотографирован, судьба не решена`, `переотмыт`, `выброшен` или `решу позже`.
7. **Журнал → Кариотип**. Из карточки фотографии или ивента — переход в импорт кариотипа.
8. **Кариотип → Импорт**. Оператор перетаскивает PSD. Программа извлекает слои → хромосомы. Каждой хромосоме сразу присваивается канонический ID `<метафаза>.cNN`.
9. **Кариотип → Разметка хромосомы**. Оператор размечает каждую хромосому: ставит центромеру, отмечает сигналы (4 типа), при необходимости — хромосомные мутации. Получается идеограмма.
10. **Кариотип → Разметка генома (метафаза)**. Оператор раскладывает хромосомы по матрице класс × субгеном. Сохраняет — формируется **кариотип метафазы**.
11. **Кариотип → Разметка генома (гибридизация)**. Оператор выбирает лучшие хромосомы из всех метафаз, при необходимости отмечает геномные мутации. Сохраняет и утверждает результат — формируется **кариотип образца**, который переводит образец в `есть результат`. Программа копирует файлы хромосом в папку кариотипа.
12. **Кариотип → Экспорт**. Оператор выбирает утверждённый кариотип образца, шаблон, режим отображения, формат (TIFF). Получает обзорное изображение.
13. **Журнал → Карточка образца**. На карточке видны все плитки кариотипов и сохранённых сравнений.

## Сценарий 2. Сравнение С Эталоном

1. **Атлас → Эталоны**. Оператор находит эталонный/теоретический кариотип нужного вида.
2. **Атлас → Сравнение**. Открывает раскладку "два кариотипа рядом". Слева — кариотип образца, справа — эталон.
3. Оператор смотрит на разницу: какие классы отличаются, где есть лишние сигналы, где аномалии.
4. **Сохранить сравнение**. Раскладка сохраняется как `<образец>.cmp.<n>` и появляется на карточке образца.

## Сценарий 3. Поиск Зависших Материалов

1. **Журнал → Прогресс**. Оператор открывает доску с пятью колонками.
2. Видит колонку `отмыт` с препаратами, которые отмыты, но не гибридизованы. Заходит в карточку зависшего препарата.
3. Решает: либо запланировать гибридизацию, либо пометить как `выброшен`.

## Сценарий 4. Заметка Без Привязки К Дате

1. **Журнал → Заметки**. Оператор пишет свободную заметку.
2. Заметка появляется в ленте текущего дня, **но не в календаре**.
3. Поиск по заметкам — отдельный экран.

## Сценарий 5. Аномалия В Кариотипе

1. **Кариотип → Разметка генома**. Оператор видит, что у образца три хромосомы класса `5D` вместо двух.
2. Оператор выбирает ячейку `5D` в матрице.
3. В правой панели → `Аномалия` → поиск по справочнику геномных мутаций.
4. Если нужного типа нет (например, "трисомия 5D" или просто "трисомия") — оператор добавляет его в справочник прямо отсюда.
5. Метка аномалии появляется рядом с ячейкой.
6. Кариотип переходит в `утверждён`, при экспорте на месте `5D` — три хромосомы со стрелкой и подписью аномалии.

## Сценарий 6. Замена Хромосомы В Утверждённом Кариотипе

1. **Кариотип → Кариотип образца** (статус `утверждён`).
2. Оператор замечает, что одна хромосома получилась плохо. Он находит лучшую в той же или другой метафазе.
3. Заменяет хромосому в кариотипе.
4. Утверждение **не отзывается** — это нормальная экспертная замена. Файл новой хромосомы копируется в папку кариотипа.
5. История замен видна в карточке кариотипа.

## Сценарий 7. Постгибридизационная Отмывка

Это **не отдельный ивент**, а решение во время или после фотографирования.

1. **Журнал → Фотографирование**. После фото препарат в статусе `сфотографирован, судьба не решена`.
2. Оператор открывает карточку препарата.
3. Решает: `переотмыт` (препарат отмывается повторно для новой гибридизации) или `выброшен`.
