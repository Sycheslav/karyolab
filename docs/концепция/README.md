# Концепция Karyolab v2

Этот раздел собран **для учёного-заказчика** и пишется простыми словами. Здесь нет кода, классов и баз данных — только то, что описывает суть программы:

- зачем она нужна;
- какими понятиями она оперирует;
- по какой логике работает каждый раздел;
- что в каком месте видит пользователь и какие решения он принимает.

Технические детали реализации (бэкенд, фронтенд, дизайн интерфейса) лежат в отдельной ветке [`docs/разработка/`](../разработка/README.md). Эта ветка — для инженеров.

## Главный Принцип

Karyolab v2 не пытается заменить эксперта. Программа берёт на себя рутину:

- хранит лабораторную историю образца от засева до фотографии;
- помогает разметить хромосомы и идеограммы;
- помогает собрать и сохранить кариотип;
- помогает сравнивать кариотипы между собой и с эталонами;
- хранит справочники, чтобы у журнала, кариотипа и атласа был один словарь.

Все научные решения остаются за оператором: какая хромосома хорошая, какой класс ей присвоить, что считать аномалией, какой кариотип взять в качестве эталона. Программа предлагает, оператор подтверждает.

Эта папка — лёгкий слой "для чтения и обсуждения". Подробные документы по каждой теме (статусы, экраны, конкретные правила) лежат в существующих папках:

- [журнал/](../журнал/README.md) — детальное описание журнала.
- [кариотип/](../кариотип/README.md) — детальное описание раздела кариотипа.
- [атлас/](../атлас/README.md) — детальное описание атласа.

В файлах концепции даются ссылки на конкретные подробные документы. Учёному не обязательно открывать их все — концепция самодостаточна. Но если нужно проверить какое-то правило в деталях, всегда есть путь.
