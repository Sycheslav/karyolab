# Фильтры И Поиск

В реальной базе быстро накапливается слишком много хромосом, чтобы показывать их все сразу. Поэтому атлас работает как поисково-фильтровая среда: эксперт сначала задаёт условия, а потом смотрит результат в одной из раскладок (см. [09_сетки_и_сравнения.md](09_сетки_и_сравнения.md)).

## Базовые Оси Фильтрации

Атлас фильтрует по тем же осям, по которым строит раскладки:

- `зонд` или `панель зондов` (см. [03_зонды_и_флюорохромы.md](03_зонды_и_флюорохромы.md));
- `вид` (см. [04_виды_и_субгеномы.md](04_виды_и_субгеномы.md));
- `субгеном`;
- `класс хромосомы` (см. [05_классы_хромосом_и_аномалии.md](05_классы_хромосом_и_аномалии.md));
- `тип аномалии`;
- `образец`;
- `препарат` или `окрашенный препарат`;
- `метафаза`;
- `источник данных`: `лабораторный`, `теоретический`, `все`;
- `статус кариотипа`: `утверждён`, `готов к проверке`, `черновик`, `архивный`.

Эти оси можно комбинировать. Любая комбинация должна корректно превращаться в условие выборки и попадать в URL экрана, чтобы фильтр можно было сохранить и поделиться им.

## Поиск По Значению

Свободный поиск работает по тем же значениям, что и таблицы кариотипа (см. [../кариотип/10_сводные_таблицы_и_поиск.md](../кариотип/10_сводные_таблицы_и_поиск.md)):

- по `id` или имени образца (`1730.25`, `добрыня`, `ae.speltoides`);
- по `виду`;
- по `имени зонда`;
- по `имени флюорохрома`;
- по `обозначению класса` (`5D`, `1RS.1BL`);
- по `типу аномалии` (`трисомия`, `замещение`);
- по `reference_label` эталона.

Поиск не должен заменять фильтры. Это быстрый способ найти конкретный объект, после чего эксперт переходит к исследованию.

## Скейл Просмотра

В атласе обязательна идея `скейла просмотра`. Без него фильтр "все хромосомы по `5D`" может вывалить сотни одинаковых хромосом и сделать экран бесполезным.

Скейл управляет тем, насколько детально показываются результаты:

- `образец` - в каждой ячейке показан один представитель класса от каждого образца;
- `метафаза` - в ячейке показаны представители каждого класса от каждой метафазы;
- `все хромосомы` - показаны все подходящие хромосомы;
- `сводка` - вместо хромосом показаны числа: сколько подходящих хромосом, сколько образцов, сколько аномалий.

Дефолт - `образец`. Это даёт краткую и быстро читаемую матрицу. Переход на `все хромосомы` - явное действие, чтобы эксперт понимал, что сейчас будет много объектов.

## Сортировка

Сортировка должна быть предсказуемой. Базовые правила:

- образцы сортируются по `id` в естественном порядке (`1730.25` идёт перед `1830.25`);
- эталоны сортируются по `reference_label`, затем по виду;
- внутри одного образца хромосомы сортируются по метафазе и порядковому номеру (`m1.c01`, `m1.c02`, `m2.c01`...);
- внутри ячейки хромосомы одного класса сортируются: сначала эталоны, потом образцы;
- классы хромосом сортируются в естественном порядке `1A, 1B, 1D, 2A, 2B, 2D, ...`, нестандартные классы (`1RS.1BL`) идут после стандартных по соответствующему номеру.

Эксперт может переопределить сортировку для конкретной раскладки, но дефолт всегда такой.

## Быстрые Пресеты

Для типовых задач удобны быстрые пресеты фильтров:

- `все эталоны вида X`;
- `все образцы с `5D` у вида X`;
- `все случаи трисомии`;
- `все кариотипы с `GAA + pAs1``;
- `новые кариотипы за месяц`;
- `все теоретические записи по виду X`;
- `все аномалии типа `замещение``.

Пресеты не зашиваются в код жёстко. Эксперт может сохранить любой текущий набор фильтров и режим как именованный пресет в своей сессии.

## Связь С Сетками

После применения фильтра атлас переходит к одной из раскладок:

- если выбран один или несколько образцов и не задан жёсткий класс - матрица класс x субгеном x образец;
- если выбраны два кариотипа - раскладка `два кариотипа рядом`;
- если выбран один класс - раскладка по одному классу у разных образцов;
- если выбран один субгеном - раскладка по одному субгеному у разных образцов;
- если выбран один зонд и много образцов - матрица класс x субгеном x образец, отфильтрованная по зонду;
- если выбран один образец и несколько препаратов или гибридизаций - раскладка `один образец, разные препараты`/`разные зонды`.

Раскладки описаны в [09_сетки_и_сравнения.md](09_сетки_и_сравнения.md).

## Граница Между Поиском И Фильтрами

Простое правило:

- поиск отвечает на вопрос `где это?`;
- фильтры отвечают на вопрос `что в выборку?`.

Поиск возвращает короткий список объектов с переходом к ним. Фильтры готовят холст исследования. Если поиск нашёл, например, эталон, эксперт может одной кнопкой перенести его в текущие фильтры (`добавить в выборку`) и перейти к раскладке.

## Что Не Делает Поиск Атласа

Поиск атласа сознательно не делает несколько вещей:

- не меняет статусы образцов и кариотипов;
- не редактирует объекты;
- не дублирует прогресс журнала (`в работе`, `есть результат`) - это задача журнала, см. [../журнал/09_прогресс_и_поиск_висяков.md](../журнал/09_прогресс_и_поиск_висяков.md);
- не подменяет таблицы кариотипа (`сколько метафаз импортировано`) - это задача кариотипа, см. [../кариотип/10_сводные_таблицы_и_поиск.md](../кариотип/10_сводные_таблицы_и_поиск.md).

Атлас отвечает только на вопрос "что у нас уже размечено и как это выглядит на фоне другого".
