# Эталонные (Теоретические) Кариотипы

В терминологии заказчика **`эталонный кариотип` = `теоретический кариотип`**. Это синонимы. Используется чаще термин "теоретический", потому что эталон в атласе — это не "правильный ответ", а опорная картинка вида/линии для сравнения с реальным размеченным кариотипом образца.

Эталон не является отдельной сущностью "эталонная хромосома" или специальным объектом "образцовый класс". Эталон — это либо особый кариотип образца с пометкой "эталонный/теоретический", либо чисто теоретическая запись из литературы или из накопленного опыта оператора.

## Принципиальное Решение

В Karyolab v2 эталон строится на этой модели:

- эталон = `кариотип образца` с включённым флагом "эталонный/теоретический", либо `теоретическая запись` (см. [07_теоретические_данные.md](07_теоретические_данные.md));
- класс хромосомы эталона = соответствующая ячейка матрицы этого кариотипа;
- набор эталонов = подсписок избранных кариотипов в атласе;
- старая логика "один эталон на класс на панель" из v1 (E1/E2) **не воспроизводится**: на один класс/субгеном может быть несколько эталонов (разные виды, разные источники).

Это нужно, чтобы не дублировать данные. Размеченный кариотип уже содержит всё, что было бы в "эталоне": классифицированные хромосомы, идеограммы, аномалии, набор зондов, происхождение.

## Поля Эталонной Метки

У объекта `кариотип образца` (`SampleKaryotype` во `frontend/src/lib/types.ts`) добавляется логический слой эталона:

- `is_reference` - флаг "это эталон";
- `reference_label` - короткое имя эталона для удобства поиска (`Чайниз Спринг`, `Безостая 1`, `H. linnaeanum 2018`);
- `reference_scope` - область применимости (`вид`, `группа видов`, `линия`, `гипотеза`);
- `reference_source` - источник: собственный лабораторный кариотип, литература, публикация;
- `reference_notes` - заметки эксперта, почему этот кариотип взят как эталон.

Эти поля можно ставить и снимать. Снять флаг "эталонный" не удаляет кариотип, он остается в кариотипе образца как обычный результат.

## Где Ставится Метка

Поставить или снять флаг "эталонный" можно в двух местах:

- в кариотипе - на карточке `SampleKaryotype` есть переключатель "сделать эталоном";
- в атласе - в карточке кариотипа можно использовать тот же переключатель.

Это удобно, потому что иногда эксперт принимает решение прямо во время работы с разметкой, а иногда - только после сравнения с другими образцами в атласе.

## Список Избранного

В атласе на экране `Эталоны` живет список избранных кариотипов, помеченных эталонными:

- список можно фильтровать по виду, набору субгеномов, набору зондов;
- список упорядочивается по `reference_label`, виду, дате;
- из карточки эталона есть быстрый переход в сравнение и в исходный кариотип образца;
- эталоны видны в панели выбора рядом с обычными образцами, но визуально отделены пометкой `эталон`.

В пикерах сравнений эталоны должны легко выбираться без длинного поиска по всему списку образцов.

## Что Можно "Вытащить" Из Эталона

Из эталонного кариотипа в любую раскладку атласа можно "вытащить" отдельные элементы для сравнения:

- одну хромосому конкретного класса (`5D` эталона);
- идеограмму этой хромосомы;
- весь субгеном (`D` эталона);
- весь кариотип целиком как столбец/строку матрицы или как одну из двух раскладок "слева/справа".

При этом исходный эталонный кариотип остается неизменным. Атлас работает с его представлением, а не с его данными.

## Импорт Эталона

Бывает ситуация, когда эталонный кариотип получен извне: из публикации, из старой системы, от коллег. В этом случае эталон создается через стандартный путь кариотипа:

- создается обычный `кариотип образца` (с пометкой "источник: литература");
- хромосомы и идеограммы либо привязываются к импортированным изображениям, либо размечаются как теоретические;
- кариотип помечается флагом "эталонный".

Если у источника нет полной лабораторной цепочки (нет образца, растения, препарата, фото, метафазы), такой эталон одновременно является `теоретической записью` (см. [07_теоретические_данные.md](07_теоретические_данные.md)) и должен явно помечаться как теоретический эталон, чтобы не путать его с лабораторным кариотипом.

## Несколько Эталонов На Один Класс

Эталонов на один и тот же класс хромосомы может быть несколько:

- один эталон вида `T.aestivum` (например `Чайниз Спринг`);
- альтернативный эталон того же вида у другой линии (`Безостая 1`);
- эталон близкого вида для сравнения;
- литературный эталон для проверки гипотезы.

Это не ошибка и не повод выбирать "главный эталон" жестко. Атлас должен поддерживать выбор одного или нескольких эталонов в раскладке. Старая идея "ровно один эталон на класс на панель" в новой системе не используется.

## Эталон В Сравнениях

В сетках и сравнениях эталоны участвуют наравне с образцами, но визуально отличаются:

- ярлык `эталон` на карточке кариотипа;
- акцентный цвет рамки в матрице (нейтральный, не красный/зеленый, чтобы не путать с каналами);
- быстрый переключатель "показать выбранный эталон рядом с каждым образцом".

Подробнее это описано в [09_сетки_и_сравнения.md](09_сетки_и_сравнения.md).

## Что Эталон Не Делает

Эталон в новой модели сознательно не делает:

- не подменяет реальные хромосомы образцов в матрице;
- не считается "правильным ответом", к которому должны сходиться все остальные;
- не редактируется атомарно "по классу" - редактируется как обычный кариотип образца;
- не блокируется от удаления только из-за статуса эталона - но при попытке удалить кариотип, который где-то выбран как эталон, должно быть явное предупреждение.
