# журнал

0\) Нужно продумать и реализовать единую систему названий объектов, чтобы в названии каждого дочернего объекта сохранялся номер исходного образца.

Предлагаемый принцип нумерации:

**1\. Образец**  
Формат: `1730.25`  
Где:  
`1730` — номер линии,  
`25` — год посева.  
Важно: номер линии может быть не только числовым, но и буквенным.

**2\. Растение**  
Формат: `1730.25.1`  
Где:  
`1730.25` — номер образца,  
`1` — порядковый номер растения в рамках образца.

Также нужен вариант для смеси растений:  
`1730.25.0`

Альтернативный более читаемый вариант:  
`PL-1730.25-1`

**3\. Препарат**  
Формат: `1730.25.1.1`  
Где:  
`1730.25.1` — номер растения,  
`1` — порядковый номер препарата для этого растения.

Альтернативный вариант:  
`SLD-1730.25.1-1`

**4\. Окрашенный препарат**  
Формат: `1730.25.1.1.1`  
Где:  
`1730.25.1.1` — номер препарата,  
`1` — порядковый номер цикла гибридизации / окрашивания.

Альтернативный вариант:  
`SLD-1-1730.25.1.1`

Главное требование: во всех названиях объектов ниже уровня образца должен сохраняться номер образца, чтобы по названию любого объекта можно было понять, к какому образцу он относится.

Также нужно предусмотреть аналогичный принцип именования для следующих сущностей:

* Фотография  
* Метафаза  
* Отдельные хромосомы  
* Идеограмма  
* Обзорный кариотип

То есть структура должна быть логичной, расширяемой и одинаково понятной для всех типов объектов.

# атлас тз первое и не повторимоое \\=

атлас представляет собой среду для анализа существующих кариотипов. 

на уровне атласа кариотип представляет собой набор размеченных по классам хромосом, их идеограммы, информацию о расположении сигналов на каждой из них. в атласе происходит сравнение конкретных хромосом на уровне кариотипов.

основное назначение атласа \- выявление полиморфизма и фиксация наследования тех или иных хромосом и их особенностей внутри геномов разных растений.

под полиморфизмом понимается естественное отклонение у разных образцов распределения паттерна окраски \- отсутствие/наличие блоков, относительное расстояние между ними и их размер. 

- фильтрация по непохожести паттерна окрашивания данным зондов среди всех образцов   
- (возможно сравнение кол-ва уникальных/отклоняющихся/повторяющихся паттернов) 

фиксация наследования \- наоборот \- поиск похожих хромосом среди образцов (стоит понимать что чаще всего “типичная” хромосома может представлять собой хромосомы не одинаковые и не родственные в узком смысле \- так что это  корректно только для непосредственно родственных образцов)

“автоматизация” на уровне атласа представляет собой систему фильтров и инструментов для сравнения и анализа материала, но не более. все выводы и предположения формирует и фиксирует оператор. возможное развитие \- системы обработки данных \- встроенные модели для сравнения абсолютных величин хромосом и относительных расстояний распределения сигналов.

возможное развитие автоматизации на основе данных атласа \- полуавтоматическое определение хромосом на уровне кариотипической разметки \- в виде предположения о возмодном классе хромосомы по идентифицированному паттерну окрашивания. 

основные объекты фигурирующие в атласе: 

хромосомы, но только в контексте состава набора кариотипа.  
кариотип образца \- сборный “красивый” кариотип собранный из лучших хромосом всех метафаз данного растения данного образца.  
кариотип метафазы \- кариотип одной метафазы данного растения данного кариотипа  
эталонный кариотип \- кариотип, собранный из литературных данных. является примером “классического” проявления сигналов данного зонда на данном виде.

для корректной работы атласа в нем необходимо сосредоточить информацию не только о конкретных кариотипах, но и знания о зондах, морфологии (возможно функциональная характеристика) хромосом и видах изучаемых растений.

зонды:   
 зонд характеризуется:   
\-название  
\-флуорохром (и канал цвета соответственно)   
\-производитель  
\-целевая последовательность  
\-комментарий

виды: возможность настраивать типовые субгеномы для видов //комментарии к видам//  
субгеномы: возможность добавления и редактирования субгеномов //комментарий и описание//

с точки зрения оператора атлас на главной странице представляет собой совокупность настраиваемых фильтров и режимов отображения кариотипов:   
выбор отображаемых файлов идет по пути: 

- выбор зонда (одного, даже если он в составе панели гибридизации),   
- выбор с поиском по списку \- одного или нескольких кариотипов.   
- в том числе выбор одного образца с разными препаратами (растениями)  
- выбор одного образца с разными зондами гибридизации.   
- выбор и сравнение по субгеному у разных образцов

отображение выбранных объектов должно быть максимально удобным для сравнения \- то есть сравниваемые хромосомы (одного класса) ВСЕГДА должны располагаться рядом (в первой итерации приложения было отличное отображение в атласе по моему в виде прокручиваемой таблицы, но чуть чуть побольше ее конечно бы сделать).

кроме того атлас предполагает наличие служебных страниц для добавления и редактирования информации о конкретных объектах атласа (зонды, субгеномы, виды), а также карточки данных объектов.

при работе в атласе никакие файлы кариотипов не изменяются и не редактируются \- происходит их просмотр. файл обзора не создается, только операции с уже существующими файлами отдельных хромосом и/или их идеограмм в составе кариотипов.   
при необходимости описать сравнение, добавить к нему комментарии или вывод должна быть возможность сформировать *отдельный* файл (хранящийся в структуре атласа, отображаемый в карточках затронутых образцов) \- сохраняющий информацию о выбранных фильтрах сравнения, режиме отображения и добавленных заметках, комментариях, выводах \- позволяющий вернуться к данному сравнению и повторно его открыть (редактирование возможно, но с сохранением всех состояний и возможностью увидеть список изменений в сравнении с момента первого сохранения).
