## Сквозные Изменения

### 0. Единая Система Именования Объектов (Naming Convention)

**Заказчик хочет:** чтобы из названия любого дочернего объекта (растения, препарата, окраски, фото, метафазы, хромосомы, идеограммы, обзора) можно было однозначно восстановить номер исходного образца. Принцип построения — иерархическое склеивание ID через точку, формируемое от номера образца.

**Канонический формат (договорённость):**

| Уровень | Шаблон | Пример | Семантика |
|---|---|---|---|
| Образец | `<линия>.<год>` | `1730.25` | Линия может быть числовой (`1730`) или буквенной (`добрыня`). Год — две цифры посева. |
| Растение | `<образец>.<n>` | `1730.25.1` | `n` — порядковый номер растения внутри образца. Для смеси растений — `0`: `1730.25.0`. |
| Препарат | `<растение>.<n>` | `1730.25.1.1` | `n` — порядковый номер препарата от этого растения / смеси. |
| Окрашенный препарат | `<препарат>.<n>` | `1730.25.1.1.1` | `n` — порядковый номер цикла гибридизации (1, 2 или 3). |
| Фотография | `<окраска>.<n>` | `1730.25.1.1.1.1` | `n` — порядковый номер фото в рамках окраски. |
| Метафаза | `<окраска>.m<n>` | `1730.25.1.1.1.m1` | `m` — префикс «метафазная пластинка». Несколько фото одной метафазы группируются здесь. |
| Хромосома | `<метафаза>.c<n>` | `1730.25.1.1.1.m1.c01` | `n` с ведущим нулём (двузначное), временное имя `c01`, при разметке заменяется на `1A`, `5D` и т. п. в `displayName`. |
| Идеограмма | `<хромосома>.idg` | `1730.25.1.1.1.m1.c01.idg` | Привязана 1‑к‑1 к хромосоме. |
| Обзорный кариотип | `<образец>.kar.<n>` | `1730.25.kar.1` | `n` — порядковый кариотип образца; `kar.1` — лицевой по умолчанию. |

**Альтернативные «читаемые» отображения** (только в UI, не в `id`):

- образец: `S-1730.25`;
- растение: `PL-1730.25-1`, для смеси `PL-1730.25-MIX`;
- препарат: `SLD-1730.25.1-1`;
- окраска: `STN-1-1730.25.1.1`;
- фото: `PHO-1-1730.25.1.1.1`.

В коде все «читаемые» формы должны строиться **из канонического `id`** через утилиту-форматтер, а не сохраняться отдельно.

**Правила**

1. `id` объектов в БД и сторе — канонический (с точками).
2. В UI всюду выводится либо канонический `id`, либо читаемая форма с тем же контентом (выбирается константой темы / настройкой). Не должно быть «вырванных» меток вроде `S-${id}` без префикса образца, как сейчас в `widgets/ProgressLifecycle.tsx:36–40` (`label: \`S-${s.id}\``) и `cards/SampleCard.tsx:121,130`.
3. При создании дочернего объекта система **всегда** склеивает `id` родителя + порядковый номер. Авто-генерация `Date.now()` (как в `forms/SlideForm.tsx:60–61`, `forms/GerminationForm.tsx:35`, `forms/WashForm.tsx:90`, `forms/HybridizationForm.tsx:75`, `forms/PhotographingForm.tsx`, `forms/FreeEventForm.tsx:32`, `forms/SampleForm.tsx`) **остаётся только для `id` ивента** (служебный объект), но не для физических объектов.
4. Ввод свободного «названия партии» (`batchName`) — отдельное поле, оно не подменяет `id`.

**Что нужно сделать в коде**

- Добавить модуль `lib/naming.ts` с функциями:
  - `nextChildId(parentId, kind, takenIds): string`;
  - `formatSampleId(id): string` (например, `S-1730.25`);
  - `formatPlantId(plant)`, `formatPreparationId(prep)` и т. д. — по таблице выше.
- Перевести генерацию `id` во всех формах создания на эти функции:
  - `forms/SampleForm.tsx:42–52` — `id` уже задаётся пользователем, оставить как есть.
  - `forms/SlideForm.tsx:60–61` — заменить `Date.now()`-генерацию на `nextChildId(plant.id, "preparation", ...)`. Если выбрана смесь — родитель `<sample>.0`.
  - `lib/store.ts:commitKaryotypeImport` (строки 440–455) — при создании `ChromosomeObject` использовать `nextChildId(metaphaseId, "chromosome", ...)` вместо `CHR-${metaphaseId}-${idx+1}`.
- В `lib/types.ts` (строки 57–104) добавить поле `displayName?: string` уже есть у хромосомы, но добавить `displayId?: string` для всех уровней; убрать поле `Sample.numericId` или зафиксировать его как «техническое» (сейчас оно нигде не используется).
- В `lib/mockData.ts` (строки 167–344) переименовать тестовые объекты согласно новой схеме (например, `P1-1818.25` → `1818.25.1`, `SLD-1818-A` → `1818.25.1.1`, `SLD-1818-MIX` → `1818.25.0.1`).
- В `widgets/ProgressLifecycle.tsx:36–40, 43–54` — выводить полный `id` (с префиксом образца), а не урезанный `prep.id`, у которого сегодня теряется связь с образцом.

**Документация** — см. блок [Изменения в документации → Именование](#именование-объектов).
