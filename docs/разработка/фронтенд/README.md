# Фронтенд Karyolab v2

Документы по фронтенду. Текущая задача — описать стек и контракты с бэкендом, не переписывать сам фронтенд.

> **Внимание:** заказчик попросил сейчас не трогать сам фронтенд. Этот раздел описывает существующее состояние и подготовительные документы для будущих доработок.

## Структура

- [01_стек_и_структура.md](01_стек_и_структура.md) — технологии, организация кода, команды разработки.
- [02_локализация_и_дизайн.md](02_локализация_и_дизайн.md) — правила локализации, палитра, типографика.
- [03_контракты_с_бэкендом.md](03_контракты_с_бэкендом.md) — какие данные фронт ждёт от бэка (общая модель, без конкретного API — конкретика в [бэкенд/03_api.md](../бэкенд/03_api.md)).
- [04_состояние_и_zustand.md](04_состояние_и_zustand.md) — где какое состояние живёт, чем разделены сторы.
- [05_роутинг_и_навигация.md](05_роутинг_и_навигация.md) — структура маршрутов и горизонтальные вкладки.

## Ключевые Решения

- **Стек:** React 18 + Vite + TypeScript + Tailwind CSS + Zustand + React Router v6.
- **Сборка:** `npm run build` → `dist/` → раздаётся через Nginx (см. `Dockerfile` в корне проекта).
- **Деплой:** Railway PaaS.
- **Локализация:** только русский в UI, никаких медицинских терминов.
- **Палитра:** болотно-зелёная.
- **Навигация:** боковое меню только для верхних разделов (`Журнал`, `Кариотип`, `Атлас`); подразделы — горизонтальные вкладки.
