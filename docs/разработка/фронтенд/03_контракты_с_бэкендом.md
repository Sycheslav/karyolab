# Контракты С Бэкендом

Этот документ описывает **что фронтенд ждёт от бэкенда** на уровне доменной модели. Конкретные эндпоинты — в [бэкенд/03_api.md](../бэкенд/03_api.md).

## Принцип Согласования

- Все типы доменной модели на фронте живут в `frontend/src/lib/types.ts`.
- Эти типы должны быть совместимы с тем, что отдаёт API.
- Изменения в модели данных идут **сначала в концепцию**, потом в `types.ts`, потом в API.

## Доменные Объекты, Которые Нужны Фронту

### Образец (Sample)

```ts
type Sample = {
  id: string;                    // "1730.25"
  name: string;
  species: SpeciesRef;
  parents?: { mother: string; father?: string };
  year: number;
  storageLocation?: string;
  status: SampleStatus;
  createdAt: string;
  notes?: string;
};

type SampleStatus =
  | "зарегистрирован"
  | "проращивается"
  | "в работе"
  | "есть результат"
  | "архивный";
```

type Plant = {

### Метафаза / Хромосома / Идеограмма

```ts
type Metaphase = {
  id: string;                    // "<okr.prep>.m<N>"
  stainedPreparationId: string;
  photoId?: string;
  importedFromPsd: boolean;
};

type Chromosome = {
  id: string;                    // "<метафаза>.cNN" — канонический, неизменяемый
  metaphaseId: string;
  filename: string;              // "untitled-1" или "..._5D.png"
  classCode?: string;            // "5D" — может отсутствовать до разметки
  hasIdeogram: boolean;
  status: ChromosomeStatus;
};

type ChromosomeStatus =
  | "новая"
  | "класс определён"
  | "есть идеограмма"
  | "кандидат в кариотип"
  | "выбрана";

type Ideogram = {
  id: string;
  chromosomeId: string;
  scale: { topY: number; bottomY: number; centromereY: number };
  signals: Signal[];
  anomalies: ChromosomeAnomalyRef[];
};

type Signal = {
  channel: "red" | "green";
  type: "small_dot" | "dot" | "large_dot" | "segment";
  position: number;
  probeId?: string;              // имя зонда — опционально
};
```

### Кариотипы И Экспорт

```ts
type MetaphaseKaryotype = {
  id: string;                    // "<метафаза>.kar.<n>"
  metaphaseId: string;
  layout: KaryotypeLayout;
};

type SampleKaryotype = {
  id: string;                    // "<образец>.kar.<n>"
  sampleId: string;
  layout: KaryotypeLayout;
  storedChromosomes: StoredChromosomeCopy[];  // КОПИИ файлов хромосом
  status: "черновик" | "утверждён" | "экспортирован" | "архивный";
  isReference?: boolean;         // эталонный/теоретический
  referenceLabel?: string;
};

type StoredChromosomeCopy = {
  originalChromosomeId: string;  // канонический ID исходной хромосомы
  fileUri: string;               // путь к локальной копии в папке кариотипа
};

type Export = {
  id: string;                    // "<образец>.exp.<n>"
  sourceId: string;              // ID кариотипа
  format: "tiff" | "xlsx" | "txt";
  templateId: string;
};

type SavedComparison = {
  id: string;                    // "<образец>.cmp.<n>" или "cmp.<n>"
  participantIds: string[];      // кариотипы образцов / эталоны
  layoutType: "matrix" | "two_side" | "multi" | "by_subgenome" | "by_class";
  filters: ComparisonFilters;
  createdAt: string;
  comment?: string;
};
```

### Справочники Атласа

```ts
type ChromosomeAnomalyType = {
  id: string;
  name: string;                  // редактируемое оператором
  description?: string;
  marker?: { color: string; shape: string };
};

type GenomeAnomalyType = {
  id: string;
  name: string;
  description?: string;
  involvedClasses?: string[];
  marker?: { color: string; shape: string };
};

type ProbeRef = { id: string; name: string; fluorochromeId: string };
// Канал сигнала определяется через fluorochromeId. В окрашенном препарате
// канал не дублируется, чтобы справочник зондов оставался единственным источником правды.
type SpeciesRef = {
  id: string;
  name: string;
  defaultSubgenomeTemplateId?: string;
  subgenomeTemplateIds: string[];    // у вида может быть несколько рабочих шаблонов
};
type SubgenomeTemplate = {
  id: string;
  name: string;
  subgenomes: string[];          // ["A", "B", "D"] и т.д.
  expectedClassesPerSubgenome: number;
  ploidy: number;
};
```

## Поведенческие Контракты

### Канонический ID Хромосомы

- Присваивается **сразу** при импорте PSD.
- **Не меняется** при переименовании файла.
- Возвращается каждым API-эндпоинтом, отдающим хромосому.

### Сохранение Кариотипа Образца

- Запрос `POST /sample-karyotypes` копирует файлы хромосом из исходных метафаз.
- В ответе фронт получает `storedChromosomes` со ссылками на копии в папке кариотипа.
- Фронт **не отображает** оригинальные файлы хромосом для утверждённого кариотипа — только копии.

### Замена Хромосомы В Утверждённом Кариотипе

- `PATCH /sample-karyotypes/:id/chromosomes` принимает замену.
- Утверждение **не сбрасывается**.
- Сервер копирует файл новой хромосомы.

### Аномалии

- Типы **редактируются**: `POST/PATCH/DELETE /atlas/anomalies/chromosome` и `/atlas/anomalies/genome`.
- При попытке удалить тип, который где-то используется, сервер возвращает 409 со списком зависимых записей.

### Календарь И Прогресс

- `GET /journal/calendar?from=...&to=...` — ивенты в диапазоне с уже агрегированными стаками.
- `GET /journal/progress` — пять списков (`созрели`, `ждёт отмывку`, `отмыт`, `гибридизован`, `есть результат`).
- Колонка `есть результат` отдаётся пагинированной (растёт постоянно).

### Поиск

- `GET /search?q=...` — глобальный поиск с фильтрами.
- Результаты группируются по типу: образцы, ивенты, кариотипы, аномалии.
