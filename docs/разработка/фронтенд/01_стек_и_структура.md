# Стек И Структура Фронтенда

## Технологии

| Компонент | Что |
|---|---|
| Фреймворк | React 18 |
| Сборщик | Vite |
| Язык | TypeScript |
| Стили | Tailwind CSS |
| Роутинг | React Router v6 |
| Состояние | Zustand |
| Линтер | ESLint |

Точные версии — в [`frontend/package.json`](../../../frontend/package.json).

## Структура Папок

Базовая структура (как сейчас в репозитории):

```
frontend/
├── src/
│   ├── components/      # переиспользуемые компоненты по доменам
│   │   ├── layout/      # Sidebar, Header, общие лейауты
│   │   ├── journal/
│   │   ├── karyotype/
│   │   ├── atlas/
│   │   └── ui/          # тонкие обёртки над Tailwind
│   ├── pages/           # экраны верхнего уровня
│   ├── lib/             # утилиты, типы, API-клиент
│   │   ├── types.ts     # все TypeScript-типы доменной модели
│   │   ├── api.ts       # обёртки над HTTP
│   │   └── stores/      # Zustand-сторы
│   ├── App.tsx
│   └── main.tsx
├── public/
├── index.html
├── package.json
├── vite.config.ts
├── tailwind.config.js
└── tsconfig.json
```

## Команды

```bash
cd frontend
npm install       # установка зависимостей
npm run dev       # локальная разработка (Vite dev server)
npm run build     # сборка в dist/
npm run preview   # локальный предпросмотр сборки
npm run lint      # линт
```

## Сборка И Деплой

Корневой `Dockerfile` (см. [`karyolab_v2/Dockerfile`](../../../Dockerfile)):

1. Стейдж `node:20-alpine` — `npm ci && npm run build`.
2. Стейдж `nginx:1.27-alpine` — копирует `dist/` и подставляет `$PORT` в конфиг Nginx.

Переменная `PORT` приходит от Railway PaaS. Локально можно запустить:

```bash
docker build -t karyolab-frontend .
docker run -p 8080:8080 -e PORT=8080 karyolab-frontend
```

## Принципы

- **TypeScript строго**: никаких `any` без явной причины. Все доменные сущности описаны в `lib/types.ts`.
- **Zustand на домен**: один стор на крупный домен (журнал, кариотип, атлас, общая навигация). Сторы не хранят данные сервера в кеше — для этого либо отдельный data-fetching слой, либо явная синхронизация.
- **Tailwind через классы**: без отдельных CSS-файлов, кроме `index.css` для базовых правил.
- **Локальная типизация форм**: каждая форма (создание ивента, разметка идеограммы) имеет собственный тип-черновик (`Draft`), отдельный от серверного типа сущности.
