# Состояние И Zustand

## Сторы Верхнего Уровня

| Стор | Что хранит |
|---|---|
| `useNavigationStore` | Активный раздел, активная вкладка, состояние свёрнутого меню. |
| `useJournalStore` | Текущая дата, фильтры календаря, активный ивент, активный образец. |
| `useKaryotypeStore` | Активная вкладка кариотипа (импорт/кариотип/экспорт), режим разметки (хромосома/геном), активная метафаза, активная хромосома. |
| `useAtlasStore` | Активная вкладка, выбранные участники сравнения, фильтры. |
| `useDictionariesStore` | Загруженные справочники атласа (виды, зонды, классы, типы аномалий). Кешируются на сессию. |

## Принципы

- **Сторы не дублируют сетевой кеш**. Если данные пришли с сервера, они держатся в стейте только пока нужны на экране. Для долгого кеша — отдельный data-fetching слой.
- **Один источник правды для активного объекта**. Например, активный образец — в `useJournalStore.activeSampleId`. Карточка образца сама загружает данные по этому ID.
- **Формы хранят черновики локально**. При создании ивента стор не используется — у формы свой локальный стейт через `useState`. После успешного сохранения форма сбрасывается, а соответствующий стор ререндерит экраны.

## Пример: Навигация И Вкладки

```ts
type NavigationState = {
  topSection: "journal" | "karyotype" | "atlas";
  karyotypeTab: "import" | "karyotype" | "export";
  atlasTab: "matrix" | "compare" | "references" | "saved" | "dictionaries";
  setTopSection: (s: NavigationState["topSection"]) => void;
  setKaryotypeTab: (t: NavigationState["karyotypeTab"]) => void;
  setAtlasTab: (t: NavigationState["atlasTab"]) => void;
};
```

Боковое меню переключает только `topSection`. Горизонтальные вкладки переключают `karyotypeTab` или `atlasTab`. URL-роуты синхронизированы с этим стейтом.

## Пример: Разметка Идеограммы

Активная идеограмма редактируется локально в компоненте `IdeogramEditor`. При сохранении вызывается `POST /chromosomes/:id/ideograms`, и стор `useKaryotypeStore` помечает хромосому как `есть идеограмма`.

Пока идеограмма редактируется — стор не трогается. Это нужно, чтобы переключение между хромосомами в списке не сбрасывало черновик случайно. Но при попытке выйти с экрана с несохранёнными изменениями показывается предупреждение.
