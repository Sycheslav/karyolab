# Экспорт Изображений

Экспорт превращает выбранный кариотип в **итоговый файл**:

- **TIFF** — изображение публикационного качества;
- **Excel/текст** — таблица данных.

## Архитектура

Экспорт — это **тяжёлая операция**, поэтому:

1. Фронт делает `POST /karyotype/exports` — задача ставится в очередь.
2. Воркер собирает файл.
3. Файл сохраняется в `exports/<id>.<ext>`.
4. Запись `exports` обновляется, фронт по `pollUrl` получает готовую ссылку.

## Изображение TIFF

### Источники

| Тип | Что собирается |
|---|---|
| `sample_karyotype` | кариотип образца (с копиями файлов хромосом) |
| `metaphase_karyotype` | кариотип метафазы |
| `reference` | эталонный (теоретический) кариотип |
| `free_table` | произвольная подборка хромосом |

### Шаблоны

| Шаблон | Что делает |
|---|---|
| `standard_overview` | один кариотип целиком |
| `comparison_two` | два кариотипа рядом |
| `multi_select` | несколько кариотипов в одной матрице |
| `by_subgenome` | один субгеном у нескольких образцов |
| `by_class` | один класс у нескольких образцов |

### Общее Правило

Во всех шаблонах:

- хромосомы одного класса всегда рядом;
- выравнивание по центромере **включено всегда**, кроме хромосом без размеченной центромеры (для них — предупреждение в файле).

### Параметры

```ts
type ExportImageParams = {
  displayMode:
    | "chromosome_only"
    | "ideogram_only"
    | "chromosome_with_ideogram_side"   // рядом
    | "chromosome_with_ideogram_below"; // сверху-снизу
  alignByCentromere: true;            // всегда true; зарезервировано на будущее
  includeProbeLabels: boolean;
  includeAnomalyLabels: boolean;
  includeSampleNames: boolean;
  includeMetaphasePath: boolean;
  customTitle?: string;
};
```

### Раскладка

Алгоритм:

1. Группируем хромосомы по классу+субгеному.
2. Для каждого класса рассчитываем bounding box, выравнивание по центромере (вертикальное смещение каждой хромосомы так, чтобы центромеры совпадали по горизонтали).
3. Раскладываем строки матрицы (классы 1..7) и столбцы (субгеномы A,B,D,...).
4. Подставляем подписи зондов цветным текстом (красный/зелёный/синий).
5. Добавляем стрелки и подписи аномалий — чтобы они **не перекрывали** хромосомы или идеограммы.

### Рендер

Кандидаты:

- **`sharp`** + Canvas/SVG → TIFF — для базовых случаев.
- **ImageMagick CLI** — для сложных композиций.
- **`canvas`** (npm `node-canvas`) — для динамических подписей и стрелок.

Финальный TIFF: 16-битный, без сжатия с потерями (LZW допустим), цветовой профиль sRGB.

```ts
async function buildKaryotypeTiff(params: ExportImageParams, layout: Layout): Promise<Buffer> {
  const canvas = createCanvas(width, height);
  const ctx = canvas.getContext("2d");
  drawBackground(ctx);
  drawClassHeaders(ctx, layout);
  drawSubgenomeHeaders(ctx, layout);
  for (const cell of layout.cells) {
    drawCell(ctx, cell, params);
  }
  drawProbeLabels(ctx, layout, params);
  drawAnomalyLabelsWithArrows(ctx, layout, params);
  return await sharp(canvas.toBuffer("image/png"))
    .tiff({ compression: "lzw", quality: 100 })
    .toBuffer();
}
```

### Никакого Ползунка Качества

Заказчик подтвердил: для TIFF **нет ползунка качества**. Всегда максимум.

## Таблица Excel/Текст

Таблица — это **не картинка**, а структурированные данные. Используем `exceljs` для Excel и обычный CSV/TSV для текста.

### Поля

Свободная таблица позволяет выбрать:

- идентификация: ID образца, имя, вид, родители, год;
- лабораторная история: число растений, препаратов, окрашенных препаратов, метафаз;
- зонды: список с цветом канала и видом сигнала;
- разметка: число хромосом, классов, идеограмм;
- аномалии: хромосомные мутации, геномные мутации;
- кариотипы: число и статусы;
- даты ивентов: засев, отмывка, гибридизация, фотографирование;
- комментарии оператора.

Поля **выбирает оператор**. Никакого фиксированного "правильного" набора нет.

### Excel

```ts
async function buildKaryotypeXlsx(rows: ExportRow[], fields: string[]): Promise<Buffer> {
  const wb = new ExcelJS.Workbook();
  const sheet = wb.addWorksheet("Karyolab");
  sheet.columns = fields.map((f) => ({ header: humanLabel(f), key: f }));
  rows.forEach((row) => sheet.addRow(row));
  return Buffer.from(await wb.xlsx.writeBuffer());
}
```

### Текст

CSV с разделителем `;` (для русской Excel-локали) или TSV. Колонки — те же поля.

## Сохранение И Доставка

1. Воркер записывает результат в `$MEDIA_ROOT/exports/<export_id>.<ext>`.
2. Обновляет запись `exports.status = "ready"`, `exports.file_path = "..."`, `exports.file_size = ...`.
3. Меняет статус кариотипа на `экспортирован` (если был `утверждён`).
4. Фронт получает `fileUri = /api/v1/files/<file_id>` и отдаёт через браузер.

## Идемпотентность

Если оператор два раза нажал "Экспортировать" — повторный запрос с тем же `Idempotency-Key` возвращает существующий `exports.id`, не запускает новую задачу.

## Ошибки

- Хромосома без центромеры — предупреждение в `exports.warnings`, но файл собирается.
- Хромосома без файла (битая ссылка) — ошибка, экспорт прерывается.
- Кариотип не утверждён — экспорт работает, но в файле проставляется водяной знак "ЧЕРНОВИК".
