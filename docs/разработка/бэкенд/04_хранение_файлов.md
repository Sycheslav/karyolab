# Хранение Файлов

В Karyolab v2 много медиа-файлов:

- **PSD** — исходники из микроскопа (десятки МБ);
- **фотографии метафаз** — JPEG/PNG (несколько МБ);
- **извлечённые хромосомы** — PNG-слои (десятки/сотни КБ);
- **копии хромосом в кариотипе образца** — PNG (те же файлы, что и слои, но в папке кариотипа);
- **TIFF-обзоры экспорта** — большие (десятки МБ).

Реляционная БД для медиа не подходит. Используем **файловое хранилище**.

## Хранилища

### Локальное Файловое (Default)

Для одиночной установки и dev-окружения:

```
$MEDIA_ROOT/
├── samples/
│   └── 1730.25/
│       ├── plants/
│       │   └── 1/
│       │       └── preparations/
│       │           └── 1/
│       │               └── stained/
│       │                   └── 1/
│       │                       ├── photo.jpg
│       │                       ├── source.psd
│       │                       └── metaphases/
│       │                           └── m1/
│       │                               └── chromosomes/
│       │                                   ├── c01.png
│       │                                   └── c02.png
│       └── karyotypes/
│           └── kar1/
│               ├── chromosomes/      ← копии файлов хромосом
│               │   ├── c01.png
│               │   └── ...
│               └── metadata.json
├── exports/
│   └── 1730.25.exp.1.tiff
└── tmp/
    └── imports/
        └── <import_id>/
```

### S3-Совместимое (Production)

Тот же путь становится S3-ключом. Бакет один (`karyolab-media`), путь внутри — как локальный.

Конфигурация:

```
MEDIA_ROOT=/var/karyolab/media       # локальное
S3_ENDPOINT=https://...              # для prod
S3_BUCKET=karyolab-media
S3_ACCESS_KEY=...
S3_SECRET_KEY=...
```

Слой `files/storage.ts` абстрагирует разницу: один интерфейс `StorageAdapter` с реализациями `LocalStorage` и `S3Storage`. Сервисы работают с интерфейсом.

## Иерархия Папок Образца

Папки повторяют иерархию ID:

| Уровень | Папка |
|---|---|
| Образец | `samples/<sample_id>/` |
| Растение | `samples/<sample_id>/plants/<plant_seq>/` |
| Препарат | `samples/<sample_id>/plants/<plant_seq>/preparations/<prep_seq>/` |
| Окрашенный | `.../stained/<stain_seq>/` |
| Метафаза | `.../metaphases/m<n>/` |
| Хромосомы метафазы | `.../metaphases/m<n>/chromosomes/c<NN>.png` |
| Кариотип образца | `samples/<sample_id>/karyotypes/kar<n>/` |
| Копии хромосом кариотипа | `samples/<sample_id>/karyotypes/kar<n>/chromosomes/` |

## Копирование Хромосом В Кариотип Образца

При сохранении/обновлении кариотипа образца:

1. Сервер транзакционно создаёт запись `sample_karyotypes` со статусом `черновик` и `storage_dir = samples/<sample>/karyotypes/kar<n>/`.
2. Создаёт каталог `<storage_dir>/chromosomes/`.
3. Для каждой выбранной хромосомы копирует файл `metaphases/<m>/chromosomes/cNN.png` в `<storage_dir>/chromosomes/cNN.png`.
4. Считает sha256 копии и пишет в `stored_chromosomes.original_file_hash`.
5. Записывает `stored_chromosomes` со ссылкой на `original_chromosome_id` и `original_chromosome_canonical_id` (на случай, если канонический ID когда-нибудь будет нужен в дампе без БД).

Если копирование падает на середине, удаляется уже созданная папка кариотипа и откатывается транзакция.

## Замена Хромосомы

`PATCH /sample-karyotypes/:id/chromosomes`:

1. Копирует файл новой хромосомы в `<storage_dir>/chromosomes/cNN.png`, **перезаписывая** старый файл.
2. Старый файл сохраняется в `<storage_dir>/chromosomes/_history/cNN_<timestamp>.png` для аудита.
3. Обновляет `stored_chromosomes` (новый `original_chromosome_id`, новый hash).
4. Пишет `audit_log` с событием `chromosome_replaced`.

## Удаление Образца

При soft-delete образца файлы **не удаляются физически**, только помечаются. Жёсткое удаление файлов — отдельная операция администратора через CLI.

При жёстком удалении кариотипа образца его папка `karyotypes/kar<n>/` удаляется полностью.

## Размеры И Лимиты

- PSD до **500 МБ** в одном файле — допустимо.
- Фото метафазы до **100 МБ**.
- Хромосома (PNG) обычно меньше 1 МБ.
- TIFF-обзор кариотипа — до 200 МБ.

В API-роутах `/karyotype/imports` ставим явные лимиты тела запроса в Fastify (`bodyLimit`).

## Кеширование

- Превью-уменьшения хромосом и кариотипов хранятся рядом: `cNN.preview.png` (256px по большей стороне).
- Превью генерируются при первом запросе и кешируются.
