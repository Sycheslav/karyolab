# Деплой И Окружения

## Окружения

| Окружение | Назначение | БД | Файлы |
|---|---|---|---|
| `dev` | локальная разработка | локальный Postgres | локальная FS |
| `staging` | проверка перед релизом, демо для заказчика | отдельный Postgres | отдельный S3-бакет |
| `prod` | боевое использование | боевой Postgres + WAL | боевой S3 + версионирование |

Конфигурация — через `.env`. Никаких прод-секретов в репозитории.

## Контейнеризация

### Текущий Dockerfile (Только Фронт)

В `karyolab_v2/Dockerfile` сейчас собирается **только фронтенд** и раздаётся через Nginx с подстановкой `$PORT`:

```
node:20-alpine (build)  →  dist/  →  nginx:1.27-alpine (runtime)
```

### Будущий Dockerfile Бэкенда

Когда появится бэкенд, добавляется отдельный `backend/Dockerfile`:

```dockerfile
FROM node:20-alpine AS deps
WORKDIR /app
COPY backend/package*.json ./
RUN npm ci

FROM node:20-alpine AS build
WORKDIR /app
COPY --from=deps /app/node_modules ./node_modules
COPY backend/ ./
RUN npm run build

FROM node:20-alpine AS runtime
WORKDIR /app
COPY --from=build /app/dist ./dist
COPY --from=build /app/node_modules ./node_modules
COPY backend/package*.json ./
COPY backend/prisma ./prisma
RUN npx prisma generate
EXPOSE 3000
CMD ["node", "dist/server.js"]
```

### Compose Для Локальной Разработки

`docker-compose.dev.yml`:

```yaml
services:
  postgres:
    image: postgres:15-alpine
    environment:
      POSTGRES_PASSWORD: dev
      POSTGRES_USER: dev
      POSTGRES_DB: karyolab
    ports: ["5432:5432"]
    volumes: ["pgdata:/var/lib/postgresql/data"]

  redis:
    image: redis:7-alpine
    ports: ["6379:6379"]

  minio:
    image: minio/minio
    command: server /data --console-address ":9001"
    environment:
      MINIO_ROOT_USER: minio
      MINIO_ROOT_PASSWORD: minio12345
    ports: ["9000:9000", "9001:9001"]
    volumes: ["miniodata:/data"]

volumes:
  pgdata:
  miniodata:
```

Бэкенд запускается локально через `npm run dev`, тяжёлые сервисы — в Docker.

## Railway

Сейчас фронт деплоится на Railway. Когда появится бэкенд:

- Отдельный сервис `karyolab-backend` с переменными `DATABASE_URL`, `REDIS_URL`, `S3_*`.
- Фронт собирается отдельно, переменная `VITE_API_BASE_URL` указывает на бэкенд.
- Postgres — Railway-плагин или внешний managed Postgres.
- Redis — Railway-плагин.
- Файлы — внешнее S3-совместимое хранилище (Cloudflare R2, Backblaze B2 или AWS S3) для удобства бэкапов и масштабирования.

## Миграции БД

- Все миграции через **Prisma Migrate**: `prisma/migrations/`.
- На старте контейнера в prod выполняется `prisma migrate deploy` — применение миграций без интерактива.
- Никаких ручных правок схемы на проде.

## Health-Чеки

| Эндпоинт | Что проверяет |
|---|---|
| `GET /health/live` | приложение запущено |
| `GET /health/ready` | БД доступна, Redis доступен, файловое хранилище доступно |

Railway / Kubernetes используют `/health/ready`.

## Логи И Мониторинг

- **Логи** — Pino → stdout. Railway собирает в свой просмотрщик.
- **Метрики** — на старте достаточно `prom-client` с минимальным набором (request count, latency, queue size). Подключение Prometheus/Grafana — по необходимости.
- **Ошибки** — Sentry (бесплатный план), DSN через переменную.

## Команды

```bash
npm run migrate          # prisma migrate dev
npm run migrate:deploy   # prisma migrate deploy
npm run lint
npm run test
npm run worker:psd       # запуск воркера PSD
npm run worker:export    # запуск воркера экспорта
```
