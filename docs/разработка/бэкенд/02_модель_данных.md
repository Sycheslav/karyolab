# Модель Данных

Схема для PostgreSQL. ORM — Prisma. Здесь приведена логическая модель; точный синтаксис миграций будет в `prisma/schema.prisma`.

## Принципы

- У каждой таблицы есть **числовой суррогатный ключ** `id_pk` (BIGSERIAL) для FK.
- Дополнительно **иерархический ID** `id` (varchar) — то, что видит пользователь и фронт.
- Все таблицы имеют `created_at`, `updated_at` (timestamptz).
- Удаления — **soft delete** (`deleted_at`) для пользовательских данных. Жёсткое удаление только для технических временных записей.
- JSON-поля используем для **гибких атрибутов** (метаданные PSD, настройки экспорта), но не для основных связей.

### `stained_preparation_probes` (Зонды Окрашенного Препарата)

| Поле | Тип |
|---|---|
| `stained_preparation_id` | bigint FK |
| `probe_id` | bigint FK → probes |

PK: `(stained_preparation_id, probe_id)`.

Канал не хранится в этой связующей таблице: он определяется через `probe -> fluorochrome -> channel`. Это важно, чтобы не расходились справочник зондов и запись окрашенного препарата. DAPI хранится как флюорохром/зонд справочника с `is_dapi = true` и автоматически относится к синему каналу.

## Атлас (Справочники)

### `species` (Виды)

| Поле | Тип |
|---|---|
| `id_pk` | bigserial PK |
| `id` | varchar(32) UNIQUE |
| `name` | text |
| `default_subgenome_template_id` | bigint FK NULL |

У одного вида может быть несколько типовых шаблонов субгеномов: нормальный, геномно-замещённый, пользовательский рабочий вариант. Поэтому основной шаблон — только дефолт для новых образцов, а полный список хранится в связующей таблице.

### `species_subgenome_templates` (Шаблоны Вида)

| Поле | Тип |
|---|---|
| `species_id` | bigint FK → species |
| `subgenome_template_id` | bigint FK → subgenome_templates |
| `is_default` | boolean |

PK: `(species_id, subgenome_template_id)`.

### `subgenome_templates` (Шаблоны Субгеномов)

| Поле | Тип |
|---|---|
| `id_pk` | bigserial PK |
| `id` | varchar(32) UNIQUE |
| `name` | text |
| `subgenomes` | text[] — `["A","B","D"]` |

| `ploidy` | int |

## Индексы И Производительность

- Все FK покрыты индексами.
- `chromosomes(id)` — UNIQUE B-tree для быстрого поиска по каноническому ID.
- `samples(deleted_at, status)` — для прогресса.
- `events(date)` — для календаря.
- `audit_log(target_type, target_pk, created_at DESC)` — для истории объекта.
