# Аутентификация И Многопользовательский Режим

## Текущее Состояние (V2)

В Karyolab v2 многопользовательский режим **не приоритет**. Заказчик подтвердил это в анкете: один-два оператора в лаборатории работают на одной установке, риск конфликтов минимален.

Поэтому в первой версии:

- **Без логина** — приложение запускается локально на машине лаборатории и сразу доступно.
- В `audit_log` поле `actor` пишется как `"local"` (или имя машины из переменной окружения `KARYOLAB_OPERATOR`, если задано).
- Все API-эндпоинты доступны без токена.

## Что Должно Быть Готово К Будущему

Чтобы потом не переписывать половину кода, закладываем:

- **Middleware `extractActor`** — даже сейчас читает `KARYOLAB_OPERATOR` из заголовка `X-Operator`, потом превратится в полноценный JWT-парсер.
- **Параметр `actor: string` во всех сервисах** — сразу принимаем, сразу пишем в `audit_log`.
- **Отделение публичных и приватных эндпоинтов** — даже сейчас `/api/v1/*` отделено от потенциального `/public/*`, чтобы добавить middleware на префикс одной строкой.

```ts
// shared/auth.ts
export interface Actor {
  id: string;          // "local", "user:123" в будущем
  displayName: string;
  roles: string[];     // в будущем
}

export function extractActor(req: FastifyRequest): Actor {
  const operatorHeader = req.headers["x-operator"] as string | undefined;
  return {
    id: operatorHeader ? `op:${operatorHeader}` : "local",
    displayName: operatorHeader ?? process.env.KARYOLAB_OPERATOR ?? "local",
    roles: ["operator"],
  };
}
```

## Когда Подключать Полноценную Авторизацию

Триггеры:

- появилось два независимых рабочих места;
- появилась общая база данных, доступная по сети;
- появилось разделение прав (например, лаборант не может удалять кариотипы, только эксперт).

Когда хотя бы одно срабатывает — добавляем:

- таблицу `users` (id, login, password_hash, role, ...);
- JWT-токены или cookie-сессии;
- middleware проверки прав на отдельных эндпоинтах.

Это **отдельный документ** будет написан перед реализацией. Сейчас в коде только подготовительные хуки.

## Конкурентное Редактирование

Для одного оператора этого достаточно, но даже сейчас имеет смысл защититься от двух открытых вкладок:

- На критичных операциях (утверждение кариотипа, замена хромосомы) добавляем `If-Match` с `updated_at`. Если на сервере объект уже изменён — 412 Precondition Failed.
- Фронт показывает понятное сообщение и предлагает обновить.
