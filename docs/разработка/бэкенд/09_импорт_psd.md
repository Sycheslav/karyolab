# Импорт PSD

Импорт PSD — это критический процесс. От него зависит вся последующая разметка хромосом. Этот документ описывает реализацию.

## Что Делает Импорт

1. Принимает PSD-файл (multipart upload).
2. Парсит структуру слоёв.
3. Определяет, какой окрашенный препарат соответствует файлу (по метаданным или по выбору оператора).
4. Извлекает каждый слой как **отдельную хромосому**.
5. Сохраняет файлы хромосом и создаёт записи в БД.
6. Возвращает список найденных слоёв для подтверждения оператором.
7. По commit'у — сохраняет всё в основные таблицы.

## Парсинг PSD

### Библиотека

Кандидаты:

- **`psd`** (npm) — JavaScript-парсер PSD. Подходит для базового извлечения слоёв.
- **`psd.js`** — старый, но рабочий.
- **ImageMagick / GraphicsMagick** через CLI — мощный, но требует установки.

Вариант по умолчанию: `psd` npm-пакет в воркере. Если возникнут проблемы со сложными PSD — перейти на ImageMagick.

### Что Извлекаем Из Слоя

Для каждого слоя:

- название слоя (часто содержит класс, например `5D`, `1A` — но не всегда);
- рендер слоя в PNG;
- bounding box на холсте PSD;
- маска слоя (если есть) — для определения размера хромосомы;
- видимость, opacity (для фильтрации служебных слоёв).

### Что Считаем "Слой-Хромосома"

Эвристики:

- слой видим (visible);
- слой не является группой;
- bounding box не равен размеру всего холста (исключаем фон);
- слой не пустой (есть непрозрачные пиксели);
- название не похоже на служебное (`background`, `bg`, `template`, `notes`).

Эвристики настраиваются. Оператор всегда может **исключить любой слой** до commit'а.

## Поток Импорта

### 1. Загрузка

`POST /karyotype/imports` (multipart):

- `file` — PSD;
- `stainedPreparationId` — куда импортируем (опционально, можно угадать по имени файла).

Сервер:

1. Сохраняет файл в `tmp/imports/<import_id>/source.psd`.
2. Создаёт запись `karyotype_imports` со статусом `parsing`.
3. Ставит задачу в очередь.
4. Отдаёт фронту `import_id` и `pollUrl`.

### 2. Парсинг (В Воркере)

```ts
async function processPsdImport(importId: string) {
  const psd = await PSD.open(`tmp/imports/${importId}/source.psd`);
  const layers = collectChromosomeLayers(psd);

  for (const [i, layer] of layers.entries()) {
    const png = await layer.exportPng();
    const tempName = `untitled-${i + 1}`;
    await saveToTmp(`tmp/imports/${importId}/preview/${tempName}.png`, png);
    layers[i].tempName = tempName;
    layers[i].previewUri = `/files/tmp/${importId}/preview/${tempName}.png`;
  }

  await db.karyotypeImports.update({
    where: { id: importId },
    data: {
      status: "ready_to_commit",
      foundLayers: layers,
    },
  });
}
```

Метаданные: можно попытаться достать из имени файла (если имя содержит `образец`, `препарат`, `гибридизация`) или из встроенных EXIF/XMP.

### 3. Превью Для Оператора

`GET /karyotype/imports/:id`:

```json
{
  "data": {
    "status": "ready_to_commit",
    "stainedPreparationGuess": { "id": "...", "confidence": "high" },
    "layers": [
      {
        "tempName": "untitled-1",
        "previewUri": "...",
        "boundingBox": { ... },
        "size_estimate": 0.42,
        "suspect": false
      }
    ],
    "warnings": [
      { "type": "unmatched_probe", "message": "В PSD упомянут зонд X, его нет в текущей гибридизации" }
    ]
  }
}
```

### 4. Commit

`POST /karyotype/imports/:id/commit`:

```json
{
  "stainedPreparationId": "...",
  "excludedLayers": ["untitled-12", "untitled-13"]
}
```

Сервер транзакционно:

1. Создаёт запись `metaphases`.
2. Для каждого не исключённого слоя:
   - присваивает канонический ID `<метафаза>.cNN`;
   - копирует PNG из `tmp/` в `samples/.../metaphases/m<n>/chromosomes/cNN.png`;
   - создаёт запись `chromosomes` с `filename: "untitled-<i>"` (если оператор не задал класс);
3. Чистит `tmp/imports/<import_id>/`.
4. Удаляет запись `karyotype_imports` (или помечает `committed_at`).
5. Возвращает фронту список созданных хромосом.

## Ошибки И Обработка

### Файл Не PSD Или Битый

- Воркер падает с `parse_error`.
- Запись `karyotype_imports.status = "failed"`, `error_message` — понятное сообщение.
- Фронт показывает: "Файл не удалось прочитать. Возможно, это не PSD или он повреждён."

### Слоёв Не Найдено

Возможно, в PSD нет хромосом или эвристика их не находит. Фронт показывает предупреждение и предлагает оператору **выбрать слои вручную** из полного списка (включая исключённые служебные).

### Конфликт Зондов

Если в PSD упомянут зонд (через имена слоёв), которого нет в текущей гибридизации — **предупреждение**, но не блокировка. Оператор подтверждает.

### Файл Уже Импортирован

Сервер хеширует PSD при загрузке. Если такой хеш уже есть в `metaphases.psd_hash`:

- предлагается открыть существующую метафазу;
- или подтвердить повторный импорт (создать другую метафазу).

## Производительность

- PSD до 500 МБ парсится в среднем 30-90 сек.
- Воркер пишет прогресс в `karyotype_imports.progress` (`%`), фронт показывает прогресс.
- Очередь работает в одном инстансе воркера (не параллелим, чтобы не насиловать диск).
