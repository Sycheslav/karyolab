# Дизайн Экспорта

Экран экспорта опирается на макет `экспорт.png`, но в финальном продукте нужно убрать медицинские placeholder-термины и сделать экран предметным для Karyolab: образцы, кариотипы, зонды, метафазы, аномалии и шаблоны обзоров.

## Цель Экрана

Пользователь должен настроить и получить итоговый файл:

- стандартный обзор одного образца;
- сравнение нескольких образцов;
- свободную таблицу;
- сводную таблицу с данными;
- рабочий или публикационный вариант.

## Рекомендуемая Компоновка

### Выбор Данных

Левая часть экрана:

- поиск образца;
- список недавно обработанных образцов;
- выбранные образцы;
- выбор конкретного кариотипа: `кариотип образца`, `кариотип метафазы`, `эталонный (теоретический) кариотип`;
- удаление лишнего из выборки.

У одного образца может быть несколько кариотипов образца (разные растения, разные гибридизации) и несколько кариотипов метафаз. Это **разные сущности** (см. [02_объекты_и_происхождение_данных.md](02_объекты_и_происхождение_данных.md)). Пользователь должен явно выбирать конкретный кариотип, а не только образец.

Эталонный (теоретический) кариотип берётся из атласа и может быть включён в обзор для сравнения.

### Выбор Шаблона

Правая верхняя часть:

- `стандартный обзор`;
- `сравнение нескольких образцов`;
- `свободная таблица`;
- пользовательские шаблоны.

Шаблон должен объяснять, для чего он нужен. Хорошо показывать мини-превью структуры.

### Конфигурация Таблицы

Правая или нижняя часть. Свободная таблица — это инструмент оператора собирать произвольные данные по образцам. Поля выбираются по необходимости:

- идентификация образца, растения, метафазы;
- число растений, препаратов, метафаз;
- зонды (с цветом канала и видом сигнала);
- число хромосом по классам;
- хромосомные аномалии (с типами из справочника атласа);
- геномные аномалии;
- комментарии оператора;
- даты ивентов из журнала (засев, отмывка, гибридизация, фотографирование).

Поля **только предметные**. Никаких `patient`, `case`, `clinical summary` и медицинских терминов.

## Настройки Изображения

Экран должен позволять выбрать:

- режим отображения хромосомы:
  - **только хромосома** (фото с фона метафазы);
  - **только идеограмма**;
  - **хромосома + идеограмма рядом**;
  - **хромосома сверху, идеограмма снизу**;
- выравнивание по центромере **включено всегда**, кроме хромосом без размеченной центромеры (с предупреждением);
- подписи зондов цветом канала (красным/зелёным), название и опционально тип сигнала;
- подписи аномалий со стрелками к участку хромосомы; стрелки и текст не должны перекрывать саму хромосому или идеограмму;
- формат файла: для изображений — **TIFF** как основной (максимальное качество, без ползунка качества); для таблиц — **Excel/текст**, не картинка.

Если часть настроек невозможна, система объясняет почему. Например, нельзя выровнять конкретную хромосому по центромере, если центромера не размечена — она показывается без выравнивания, появляется предупреждение.

## Готовность К Экспорту

Перед большой кнопкой генерации нужен блок готовности:

- все обязательные данные есть;
- есть предупреждения;
- есть неполные классы;
- есть хромосомы без идеограмм;
- есть неподтвержденные аномалии;
- экспорт будет рабочим черновиком или финальным файлом.

Кнопка должна называться по действию: `Собрать обзор`, `Собрать сравнение`, `Экспортировать таблицу`.

## Сохранение Шаблона

Если пользователь настроил полезную конфигурацию, ее можно сохранить как шаблон:

- название;
- тип;
- выбранные поля;
- настройки подписей;
- режим хромосом и идеограмм;
- формат вывода.

Шаблон не должен сохранять конкретные образцы, если пользователь явно не выбрал это как пресет для повторной серии.

## Что Улучшить В Макете

В макете хорошая идея: выбор образцов слева, шаблон и поля справа, большая кнопка генерации. Улучшить нужно:

- заменить все медицинские слова на термины Karyolab;
- добавить выбор конкретного кариотипа у образца (`кариотип образца` / `кариотип метафазы` / `эталонный`);
- добавить предпросмотр или хотя бы схему результата;
- показать готовность и блокирующие ошибки (например, нет идеограммы у выбранной хромосомы);
- разделить экспорт изображения (TIFF) и экспорт таблицы (Excel/текст);
- дать действия после генерации: `открыть в папке`, `сохранить как`, `повторить с другим шаблоном`;
- в подписи аномалии чётко показывать стрелку к месту на хромосоме и не перекрывать сам объект текстом.
