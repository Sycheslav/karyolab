# Дизайн Разметки Хромосом

Экран создания идеограмм опирается на макет `кариотип- размертка хромосом.png`. Удачная идея макета - трехчастная работа: выбрать хромосому, увидеть оригинал и разметить шкалы, сразу увидеть итоговую идеограмму.

## Цель Экрана

Пользователь должен разметить одну хромосому так, чтобы:

- была задана длина;
- была поставлена центромера;
- были описаны красные и зеленые сигналы;
- были отмечены аномалии;
- была сохранена цифровая и графическая идеограмма.

## Рекомендуемая Компоновка

### Левая Панель

Левая панель содержит:

- переключатель уровня `метафаза / гибридизация`;
- список хромосом;
- миниатюру каждой хромосомы;
- имя или временное имя;
- индикатор наличия идеограммы;
- статус уверенности.

Список должен позволять быстро пройти все хромосомы подряд. Полезны фильтры: `без идеограммы`, `сомнительные`, `выбранные`, `исключенные`.

### Центральная Область

Центр лучше разделить на две части:

- оригинальная хромосома;
- шкалы разметки.

Оригинал нужен крупнее миниатюры, но не должен занимать весь экран. Важнее, чтобы шкалы были точными и удобными.

### Шкалы

Все названия в русском интерфейсе:

- `размер и центромера` — **одна шкала с тремя ползунками** (верхняя точка размера, нижняя точка размера, центромера). Размер определяется полуавтоматически по маске слоя; ползунки можно двигать вручную;
- `красный канал` — несколько сигналов;
- `зелёный канал` — несколько сигналов;
- `аномалии` — отдельная линия рядом со шкалами сигналов, всегда показывается, даже если аномалий нет.

Шкалы расположены рядом с изображением хромосомы и имеют тот же масштаб по высоте. При изменении масштаба изображение и шкалы остаются ровно друг с другом — это **принципиально важно**, иначе разметка теряет смысл.

Сигнал на шкале имеет один из четырёх типов:

- `маленькая точка`;
- `точка`;
- `большая точка`;
- `отрезок`.

Никаких других типов в дизайне быть не должно. На итоговой идеограмме сигнал визуально показывается как **две точки рядом на одном уровне** (или два отрезка для типа `отрезок`) — так же, как сигнал виден на фотографии.

Пользователь добавляет точки и отрезки кликом или перетаскиванием. Для каждого элемента можно редактировать тип, размер, подпись и (опционально) имя зонда, который дал этот сигнал. Имя зонда — необязательное поле; оператор не должен учить весь справочник зондов наизусть, чтобы что-то сохранить.

Для аномалий: при добавлении аномалии открывается поле с поиском по существующим типам в справочнике атласа и кнопкой добавить новый тип, если его нет в справочнике. Программа не должна предлагать предзаготовленный список — это решение оператора.

### Правая Панель

Правая панель показывает результат и действия:

- блок предпросмотра: реальная фотография хромосомы и сформированная идеограмма рядом — оператор сравнивает, насколько идеограмма соответствует фото;
- кнопка `Добавить в избранное по образцу` — пометка особенно красивой/нужной хромосомы для быстрого доступа в атласе;
- селектор `Класс и субгеном хромосомы` — оператор может задать сразу или оставить пустым, тогда хромосома сохранится без класса (имя файла останется `untitled-N`);
- кнопка `Сохранить идеограмму` — основное действие;
- короткая сводка: центромера есть, сигналов красных N, зелёных N, аномалий N.

Если оператор задал класс и субгеном, сохранение идеограммы меняет имя файла с `untitled-N` на имя, содержащее образец, субгеном и номер класса (например, `1730.25.1.1.1.m1.c01_5D.png`). Канонический ID хромосомы (`<метафаза>.cNN`) при этом не меняется. Если класс не выбран, файл остаётся без классификационного имени до этапа разметки генома.

Идеограмма обновляется при изменениях, но сохранение явное — оператор должен нажать `Сохранить`.

## Что Улучшить В Макете

Макет показывает шкалы как график. В реализации важно не сделать их просто декоративной картинкой. Это редактор данных.

Стоит улучшить:

- дать точное редактирование выбранного сигнала в панели свойств;
- добавить undo для последнего действия;
- показывать несохраненные изменения;
- не смешивать масштаб изображения и нормализованные координаты;
- явно показывать, какая хромосома выбрана;
- добавить переход `следующая без идеограммы`.

## Сохранение

Кнопка сохранения должна записывать:

- цифровую разметку;
- графическую идеограмму;
- статус хромосомы;
- дату и пользователя, если будет учет;
- комментарии и аномалии.

После сохранения левая панель должна обновить индикатор у хромосомы.
