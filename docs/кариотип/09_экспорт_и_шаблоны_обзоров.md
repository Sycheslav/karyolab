# Экспорт И Шаблоны Обзоров

Экспорт собирает рабочие и публикационные изображения кариотипа из уже размеченных хромосом. Это не ручная сборка в пустом холсте, а генерация по данным: выбранные хромосомы, классы, субгеномы, идеограммы, подписи и аномалии.

## Основные Цели Экспорта

Экспорт нужен, чтобы:

- получить обзорный кариотип в высоком качестве;
- быстро собрать сравнение нескольких образцов;
- показать аномалии и подписи;
- выгрузить таблицы по образцам и метафазам;
- сохранить результат в папку экспорта;
- при необходимости открыть файл в папке или сохранить как.

## Типы Шаблонов

### Стандартный

Стандартный шаблон показывает один кариотип:

- субгеномы;
- классы;
- выбранные хромосомы;
- идеограммы, если включены;
- подписи зондов;
- аномалии.

Это основной шаблон для одного образца.

### Мультивыбор

Мультивыбор нужен для сравнения нескольких геномов или образцов.

Логика:

- пользователь выбирает несколько образцов или кариотипов;
- столбцы могут быть субгеномами;
- внутри субгенома идут подстолбцы по образцам;
- строки соответствуют классам хромосом.

Этот режим полезен для сравнения родственников, линий, повторных гибридизаций или разных зондов.

### Свободная Таблица

Свободный шаблон - настраиваемая таблица. Он нужен, когда стандартная сетка не подходит:

- нестандартные субгеномы;
- фокус на одной аномалии;
- сравнение нескольких вариантов одного класса;
- обзор только выбранных хромосом;
- рабочая иллюстрация для обсуждения.

## Что Может Показывать Изображение

Для хромосом должны быть режимы:

- `хромосомы без идеограмм`;
- `хромосомы с идеограммами`;
- `только идеограммы`.

Выбор зависит от цели. Фото показывает реальность, идеограмма делает паттерн сравнимым, а режим `только идеограммы` полезен для атласа и схем.

## Выравнивание

В итоговом изображении хромосомы должны выравниваться по центромере. Это важно для визуального сравнения плеч и сигналов.

Если у хромосомы нет центромеры, система должна:

- предупредить пользователя;
- предложить разметить центромеру;
- либо разрешить экспорт с пометкой, что выравнивание для этой хромосомы неполное.

## Подписи

На экспортном изображении должны поддерживаться:

- название образца;
- для уровня метафазы - путь или краткая цепочка до метафазы и координаты;
- зонды гибридизации;
- цвет текста зонда по каналу сигнала;
- заголовки субгеномов;
- заголовки классов;
- подписи аномалий;
- стрелки к месту аномалии;
- подписи образцов под геномом при мультиобразцовом сравнении.

У самих хромосом не нужно дублировать класс, если он уже показан в шапке таблицы.

## Проверка Перед Экспортом

Перед генерацией система должна проверить:

- выбран ли образец или набор образцов;
- выбран ли шаблон;
- есть ли размеченные классы;
- есть ли обязательные подписи;
- есть ли центромеры для выравнивания;
- какие аномалии включены;
- какой режим отображения выбран.

Если данных не хватает, экспорт должен объяснять, что именно мешает.

## Результат

После экспорта пользователь должен получить:

- файл изображения;
- запись об экспорте;
- ссылку на исходный кариотип;
- возможность `открыть в папке`;
- возможность `сохранить как`;
- возможность повторить экспорт с тем же шаблоном.

Каталог экспорта должен быть отдельным от исходных PSD и рабочих хромосом.
