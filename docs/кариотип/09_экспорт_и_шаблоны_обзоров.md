# Экспорт И Шаблоны Обзоров

Экспорт собирает рабочие и публикационные изображения кариотипа из уже размеченных хромосом. Это не ручная сборка в пустом холсте, а генерация по данным: выбранные хромосомы, классы, субгеномы, идеограммы, подписи и аномалии.

## Что Можно Выбрать К Экспорту

Источниками экспорта могут быть:

- образцы (с возможностью выбрать конкретный кариотип образца, если их несколько);
- кариотипы метафаз;
- кариотипы образца;
- эталонные кариотипы (см. [../атлас/06_эталонные_кариотипы.md](../атлас/06_эталонные_кариотипы.md)).

Различие двух типов кариотипов в интерфейсе должно быть явным: метафазный кариотип — из одной метафазы, кариотип образца — сборный из лучших хромосом нескольких метафаз.

Для выбранного кариотипа должны быть переходы:

- открыть отображение в атласе;
- открыть редактирование в разметке кариотипа.

## Основные Цели Экспорта

Экспорт нужен, чтобы:

- получить обзорное изображение кариотипа образца или кариотипа метафазы в высоком качестве;
- быстро собрать сравнение нескольких образцов;
- показать аномалии и подписи;
- выгрузить таблицы по образцам и метафазам;
- сохранить результат в папку экспорта;
- при необходимости открыть файл в папке или сохранить как.

## Типы Шаблонов

### Стандартный

Стандартный шаблон показывает один кариотип:

- субгеномы;
- классы;
- выбранные хромосомы;
- идеограммы, если включены;
- подписи зондов;
- аномалии.

Это основной шаблон для одного образца.

### Мультивыбор

Мультивыбор нужен для сравнения нескольких геномов или образцов.

Логика:

- пользователь выбирает несколько образцов или кариотипов;
- столбцы могут быть субгеномами;
- внутри субгенома идут подстолбцы по образцам;
- строки соответствуют классам хромосом.

Этот режим полезен для сравнения родственников, линий, повторных гибридизаций или разных зондов.

### Свободная Таблица

Свободный шаблон — настраиваемая таблица. Главная задача шаблона — обеспечивать **свободу**: оператор сам настраивает строки и колонки. Никаких заранее зашитых конфигураций.

Свободная таблица также нужна, чтобы выбирать произвольную информацию для сводки и распечатки. Пример: оператор хочет выбрать все образцы, у которых среди родителей есть `1730.25`, и выписать списком, какие у каждого аномалии. Это та задача, которую обычная сетка не решает, а свободная таблица решает.

Полезные применения:

- нестандартные субгеномы;
- фокус на одной аномалии;
- сравнение нескольких вариантов одного класса;
- обзор только выбранных хромосом;
- рабочая иллюстрация для обсуждения.

Не нужно придумывать в документации лишних натянутых сценариев свободной таблицы — её ценность именно в свободе настройки.

## Общее Правило Шаблонов

Во всех шаблонах хромосомы одного класса всегда стоят рядом друг с другом. Пары хромосом одного образца стоят рядом. Хромосомы разных образцов в рамках одного класса стоят рядом, разделённые по субгеномам. Выравнивание по центромере применяется во всех шаблонах.

## Что Может Показывать Изображение

Для хромосом должны быть режимы:

- `хромосомы без идеограмм`;
- `хромосомы с идеограммами`;
- `только идеограммы`.

Выбор зависит от цели. Фото показывает реальность, идеограмма делает паттерн сравнимым, а режим `только идеограммы` полезен для атласа и схем.

## Выравнивание

В итоговом изображении хромосомы выравниваются по центромере во всех случаях, **кроме** хромосом, у которых центромера не отмечена. Это правило по умолчанию, без отдельной настройки.

Если у хромосомы нет центромеры, система должна:

- предупредить пользователя;
- предложить разметить центромеру;
- либо разрешить экспорт с пометкой, что выравнивание для этой хромосомы неполное.

## Подписи

На экспортном изображении должны поддерживаться:

- название образца (формат `Кариотип образца от растения {n}`, пример `1730.25.kar.1 от 1730.25.1`);
- для уровня метафазы — путь до метафазы и координаты, если они есть в имени файла;
- зонды гибридизации;
- цвет текста зонда по каналу сигнала (красный канал — красным текстом, зелёный — зелёным);
- заголовки субгеномов;
- заголовки классов;
- подписи аномалий;
- стрелки к месту аномалии;
- подписи образцов под геномом при мультиобразцовом сравнении.

У самих хромосом не нужно дублировать класс, если он уже показан в шапке таблицы.

Подписи аномалий не должны перекрывать хромосому или идеограмму. Это аккуратная стрелочка к месту аномалии и подпись рядом со стрелочкой (маленькая, удалённая на безопасное расстояние). Если подпись мешает, оператор может её отключить и добавить позже вручную в TIFF.

## Проверка Перед Экспортом

Перед генерацией система должна проверить:

- выбран ли образец или набор образцов;
- выбран ли шаблон;
- есть ли размеченные классы;
- есть ли обязательные подписи;
- есть ли центромеры для выравнивания;
- какие аномалии включены;
- какой режим отображения выбран.

Если данных не хватает, экспорт должен объяснять, что именно мешает.

## Форматы И Качество

Для экспорта изображения:

- основной формат — **TIFF** максимального качества (для печати и публикаций);
- PNG и PDF не являются обязательными форматами и могут быть добавлены позже как опции, но не дефолт;
- качество всегда максимальное (без слайдера качества — оператор не должен тратить время на этот выбор).

Для экспорта таблицы:

- основной формат — **Excel** (`.xlsx`) или текстовый документ (`.csv`/`.txt`);
- экспорт сводной таблицы в виде картинки **не используется** — таблица всегда уходит в Excel/текст;
- оператор сам выбирает поля таблицы из доступного списка (см. ниже).

## Поля Таблицы Данных

В таблицу можно включить поля по выбору оператора:

- образец;
- вид;
- год;
- родители;
- количество растений;
- количество препаратов;
- конкретное растение, если выбрано;
- количество метафаз;
- зонды гибридизации;
- количество хромосом;
- количество аномалий;
- список аномалий текстовой ячейкой.

## Результат

После экспорта пользователь должен получить:

- файл изображения (TIFF) или таблицы (XLSX/CSV);
- запись об экспорте;
- ссылку на исходный кариотип;
- кнопку `Открыть в папке`;
- кнопку `Сохранить как`;
- возможность повторить экспорт с тем же шаблоном.

Каталог экспорта отделён от исходных PSD и рабочих хромосом — это разные папки.
