# Разметка Генома

Разметка генома - режим, где пользователь распределяет извлеченные хромосомы по субгеномам и классам. Это переход от отдельных объектов к кариотипной картине образца.

## Основная Матрица

Базовая форма матрицы:

- столбцы - субгеномы;
- строки - классы хромосом;
- в ячейках - одна, две или больше хромосом соответствующего класса.

Для классической пшеницы удобный дефолт:

- субгеномы `A`, `B`, `D`;
- классы `1` ... `7`.

Но это только стартовая настройка. В системе должны быть другие субгеномы: `R`, `U`, `M` и любые добавленные пользователем.

## Гибкая Модель Субгеномов

Нельзя навсегда зашивать `A/B/D x 7` как единственный вариант. В реальной работе встречаются:

- диплоиды;
- тетраплоиды;
- гексаплоиды;
- неполные геномы;
- замещения между субгеномами;
- чужеродные хромосомы;
- образцы с текстовыми названиями и нестандартной историей.

Пользователь должен иметь возможность выбрать или настроить набор субгеномов для образца. Дефолт должен помогать, но не ограничивать научную работу.

## Уровень Метафазы

На уровне `метафаза` пользователь раскладывает хромосомы из одной метафазной пластинки.

Этот режим нужен, чтобы:

- понять, насколько метафаза пригодна;
- вручную назначить классы;
- увидеть нехватки, лишние объекты и сомнительные хромосомы;
- сохранить разметку конкретной метафазы;
- подготовить данные для автоматической раскладки на уровне гибридизации.

Для уровня метафазы допустимо дублировать класс в имени файла, но структурированные данные должны оставаться главным источником правды.

## Уровень Гибридизации

На уровне `гибридизация` пользователь видит хромосомы из нескольких метафаз одного окрашенного препарата или цикла.

Этот режим нужен, чтобы:

- выбрать лучшие хромосомы из разных метафаз;
- сравнить несколько кандидатов одного класса;
- собрать кариотип образца не из одной пластинки, а из лучших представителей;
- учесть повторную гибридизацию и дополнительные зонды;
- сформировать результат, который можно показать в карточке образца.

Если на уровне метафаз классы уже назначены, система может автоматически разложить хромосомы по ячейкам матрицы. Но пользователь должен видеть источник каждой хромосомы и иметь возможность заменить выбранный объект.

## Auto-Sort

`Auto-sort` - это не окончательное решение, а помощник.

Он может:

- разложить хромосомы по уже назначенным классам;
- сгруппировать кандидатов;
- подсветить пустые ячейки;
- подсветить ячейки с лишним количеством хромосом;
- предложить лучший объект по качеству или наличию идеограммы.

Он не должен:

- без подтверждения менять научный класс;
- скрывать сомнительные объекты;
- удалять лишние хромосомы;
- считать нестандартный набор ошибкой без контекста.

## Фон И Видимость

Для матрицы генома полезен черный или очень темный рабочий холст: синие тела хромосом и цветные сигналы лучше читаются на темном фоне.

При этом элементы управления должны оставаться достаточно контрастными:

- выбранная ячейка;
- подсветка аномалии;
- бейджи `трисомия`, `нуллисомия`, `замещение`;
- прогресс классификации;
- кнопки сохранения и генерации кариотипа.

## Результат Разметки

После сохранения разметки система должна знать:

- какие хромосомы к какому классу относятся;
- какие объекты выбраны как лучшие;
- какие ячейки неполные или нестандартные;
- какие аномалии зафиксированы;
- можно ли сформировать кариотип образца;
- какие данные готовы для экспорта и атласа.
