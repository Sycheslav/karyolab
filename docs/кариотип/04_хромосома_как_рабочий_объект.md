# Хромосома Как Рабочий Объект

Отдельная хромосома - базовая единица раздела `Кариотип`. Итоговый обзор важен, но строится он из объектов-хромосом, у каждой из которых есть происхождение, изображение, разметка и экспертный статус.

## Почему Не Только Картинка

Если хранить только готовый кариотип, система не сможет ответить на вопросы:

- из какой метафазы взята конкретная хромосома;
- была ли она выбрана как лучший представитель класса;
- какие сигналы на ней размечены;
- была ли у эксперта неуверенность;
- встречался ли похожий паттерн в другом образце.

Поэтому хромосома должна быть не файлом без контекста, а объектом данных.

## Что Хранит Хромосома

Минимальный набор:

- ID хромосомы;
- связь с метафазой;
- связь с исходным PSD и слоем;
- путь к изображению;
- путь к маске;
- размер по маске в пикселях;
- временное имя после импорта;
- ручной класс после разметки;
- субгеном;
- номер класса от 1 до 7 или другое значение для нестандартного случая;
- ручные рабочие флаги статуса;
- флаг наличия идеограммы;
- комментарий эксперта.

Дополнительные данные:

- качество объекта;
- причина исключения из кариотипа;
- принадлежность к выбранному кариотипу образца или кариотипу метафазы;
- ссылки на аномалии;
- история изменений класса.

## Канонический Идентификатор И Класс

После импорта каждая хромосома получает канонический ID по общей системе именования (см. [журнал/02_объекты_и_связи.md](../журнал/02_объекты_и_связи.md#система-именования)): `<метафаза>.c<NN>`. Например, для образца `1730.25`, растения `1`, препарата `1`, окраски `1`, метафазы `m1` хромосомы будут называться `1730.25.1.1.1.m1.c01`, `1730.25.1.1.1.m1.c02` и так далее. ID на этом этапе - двузначный порядковый номер, потому что класс ещё не определён.

После ручной разметки у хромосомы появляется разметочное имя класса (`1A`, `5D`, `1RS.1BL` и тому подобное). Это `displayName` - удобное представление, которое показывается пользователю и используется в названиях экспортных файлов. Канонический ID при этом не меняется: он остаётся стабильной точкой ссылки на этот объект, а класс хранится отдельным полем.

Правила:

- канонический ID никогда не меняется после импорта;
- разметочное имя (`displayName`) можно править;
- название файла на диске может включать оба компонента: `1730.25.1.1.1.m1.c01_1A.png`;
- по любому имени файла или ID должно быть видно, к какому образцу, растению, препарату, окраске и метафазе относится хромосома.

## Статусы

Для рабочей хромосомы достаточно компактного набора статусов:

- `новая` — извлечена из PSD, ещё не размечена;
- `класс определён` — назначен субгеном и номер класса;
- `есть идеограмма` — создана разметка сигналов;
- `кандидат в кариотип` — оператор вручную пометил хромосому как удобную для выбора в кариотип образца;
- `выбрана` — включена в кариотип (метафазы или образца).

Статусы `класс определён` и `есть идеограмма` не взаимоисключают друг друга — у одной хромосомы могут быть оба одновременно. Они отражают разные аспекты разметки: первый говорит про геномную классификацию, второй — про сигналы и идеограмму. Поэтому в UI индикаторы у этих статусов независимые.

Дополнительные более тонкие квалификаторы (`сомнительная`, `исключённая`, `в работе`) не нужны как отдельные статусы — это решает оператор через комментарий или просто не выбирает хромосому в кариотип. Софт не должен подменять собой экспертное решение. Даже статус `кандидат в кариотип` не выставляется автоматически: это только ручная пометка оператора.

## Уровни Просмотра

Хромосомы можно смотреть на двух уровнях:

- `метафаза` - все хромосомы одной метафазной пластинки;
- `гибридизация` - хромосомы из нескольких метафаз одного окрашенного препарата или цикла гибридизации.

Уровень `метафаза` нужен для первичной разметки. Уровень `гибридизация` нужен, чтобы выбирать лучшие представители из разных метафаз и собирать кариотип образца.

## Хороший Рабочий Объект

Хромосома считается хорошим кандидатом, если:

- видна форма;
- можно оценить длину;
- можно поставить центромеру;
- сигналы читаются;
- нет критичного наложения;
- она связана с правильной метафазой;
- у неё понятный класс или хотя бы гипотеза класса.

Это решение полностью на стороне оператора. Софт не должен помечать хромосомы автоматически как «хороший / плохой кандидат» — пометок такого вида в системе нет. Если хромосома частично плохая, оператор просто не включает её в кариотип; при необходимости можно оставить комментарий.

## Связь С Идеограммой

Идеограмма создается для конкретной хромосомы. После сохранения в интерфейсе рядом с хромосомой должен появиться индикатор: например значок, плюс или бейдж `идеограмма`.

Этот индикатор важен для работы с большим набором объектов: пользователь должен быстро понимать, какие хромосомы уже описаны, а какие еще ждут разметки.
