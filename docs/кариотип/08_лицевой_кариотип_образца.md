# Лицевой Кариотип Образца

Лицевой кариотип - итоговая, наиболее понятная и научно полезная сборка по образцу. Он может быть собран не из одной идеальной метафазы, а из лучших хромосом, выбранных из нескольких метафаз одной гибридизации или серии связанных гибридизаций.

## Зачем Он Нужен

В реальной работе не каждая метафаза идеальна. В одной хорошо видна часть классов, в другой лучше лежат другие хромосомы, в третьей подтверждается сомнительный сигнал.

Лицевой кариотип позволяет:

- выбрать лучшие представители;
- показать образец в понятной таблице;
- сохранить происхождение каждой выбранной хромосомы;
- подготовить материал для отчета, статьи или обсуждения;
- показать результат на карточке образца в журнале.

## Источники Хромосом

Кариотип образца может включать:

- хромосомы из одной метафазы;
- хромосомы из нескольких метафаз одного окрашенного препарата;
- хромосомы из разных гибридизаций, если это научно оправдано;
- дополнительные данные с другими зондами.

При этом у каждой выбранной хромосомы должна сохраняться ссылка на исходную метафазу.

## Выбор Лучших Хромосом

Пользователь выбирает лучшие хромосомы по критериям:

- читаемая форма;
- понятная центромера;
- хорошие сигналы;
- отсутствие критичных наложений;
- наличие идеограммы;
- уверенный класс;
- полезность для подтверждения аномалии.

Система может предлагать кандидатов, но пользователь утверждает финальный набор.

## Сохранение Результата

После сохранения лицевого кариотипа система должна:

- создать запись кариотипа образца;
- сохранить состав выбранных хромосом;
- сохранить порядок в матрице;
- сохранить аномалии и комментарии;
- создать папку или логическую группу результата;
- обновить статус образца на `есть результат`;
- сделать результат доступным для экспорта и карточки образца.

Если кариотипов несколько, пользователь должен выбрать, какой из них основной для карточки образца.

## Несколько Кариотипов У Одного Образца

У одного образца может быть больше одного результата:

- разные наборы зондов;
- разные гибридизации;
- отдельный обзор для одной аномалии;
- сравнение с родителями;
- рабочая версия и публикационная версия.

Поэтому `есть результат` не должен означать, что работа закрыта навсегда. Это означает, что есть хотя бы один подтвержденный результат, который можно показать и использовать.

## Статусы Кариотипа

Для результата нужны статусы:

- `черновик` - пользователь собирает набор;
- `неполный` - не все классы закрыты или есть сомнения;
- `готов к проверке` - набор собран, но не утвержден;
- `утвержден` - можно показывать на карточке образца;
- `экспортирован` - по нему создан обзорный файл;
- `архивный` - старая версия результата.

Статусы помогают не путать рабочие версии с финальными.

## Связь С Журналом

После утверждения хотя бы одного кариотипа журнал получает обратную связь:

- образец переходит из `в работе` в `есть результат`;
- на карточке образца появляется обзорный кариотип;
- из журнала можно открыть раздел `Кариотип` уже в контексте этого образца;
- в прогрессе образец больше не висит как неразобранный.

Журнал показывает результат, но не дублирует инструменты сборки.
